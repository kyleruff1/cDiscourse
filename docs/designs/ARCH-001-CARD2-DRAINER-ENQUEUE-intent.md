## Doctrine (HARD)
Classifiers NEVER gate submission acceptance. The deterministic rules engine stays the acceptance gate; classifiers run AFTER an argument is stored. Submit stays NONBLOCKING. The queue makes this *more* true: a smoke-routed submit only ENQUEUES (a fast local INSERT) then returns; all provider work moves to the background drainer.

## Scope (smoke-only; NO production-wide cutover)
IN:
1. **Enqueue routing** in `submit-argument` — a `shouldRouteToQueue(argument, debate)` predicate at the I14 dispatch point (the existing `dispatchAutoTriggerForArgument` + `EdgeRuntime.waitUntil` site, ~`index.ts:787`). **Default DISABLED** (returns false for everything → ordinary submits keep the current direct dispatch UNCHANGED). Routes to the queue **iff smoke-tagged** (the synthetic smoke prefix prior smokes use, or an explicit metadata marker). When routed: call `enqueue_classifier_job(arg_id, debate_id, family, run_mode, schema_version)` for each production family A–G (one multi-row INSERT; `ON CONFLICT` idempotent via index #5) INSTEAD of direct dispatch. Mutually-exclusive `if/else` — never both (the §A.11 double-dispatch proof; index #4/#5 are the DB backstops).
2. **Two-channel invocation (§A.2)** — (a) **Enqueue-kick** (latency path): a statement-level `AFTER INSERT` trigger on `argument_machine_observation_runs` that fires `net.http_post` to the drainer URL ONCE per statement (a batch enqueue = one INSERT = one kick), guarded by `pg_try_advisory_xact_lock(<kick_key>)` so concurrent submits coalesce. (b) **Periodic safety tick**: a `cron.schedule` at the locked **60s** interval calling `net.http_post` to the same drainer URL. Both read URL + service secret from **Supabase Vault (`vault.decrypted_secrets`)** — NEVER hardcoded (doctrine §6); duplicate tick+kick are harmless (single-flight collapses them to one drain). **Sequencing:** the `cron.schedule` is written-not-applied and must be applied/scheduled AFTER the drainer is deployed + the Vault secret is seeded (§A.11 step 1) — scheduling it before would `net.http_post` a null URL. Since only smoke-tagged args enqueue (item 1), an active tick only ever drains smoke jobs.
3. **Drainer Edge Function** (`supabase/functions/classifier-drainer/`, `verify_jwt=false` in `config.toml`, validates the shared secret internally): `acquire_drain_lease` (single-flight) → `reclaim_stale_leases` FIRST → bounded-batch loop [`claim_classifier_jobs` → classify via the existing MCP adapter (drainer timeout **≥30s**, NOT the 15s submit-path value — §A.6) → on success `finalize_classifier_job(...,'succeeded','success',…,observations)`, on terminal failure `finalize_classifier_job(...,'failed_terminal'|'dead_letter','failed',…)`, on retryable failure a run-row-only UPDATE to `retry_scheduled` (set `available_at`=now()+backoff, NOT finalize)] → `release_drain_lease`. Bounded by **C=3** provider concurrency AND **T=90s** wall-clock AND a batch-size cap; exit cleanly before the 150s Edge ceiling.
4. **`findExistingRun` fix** (`autoTriggerDispatcher.ts:189`) — make it ACTIVE-QUEUE AWARE: a cell with `state IN ('pending','leased','retry_scheduled')` is already-in-flight; do NOT re-dispatch (today it checks only `status='success'` + orders `started_at DESC`, so it mis-reads a pending `status NULL` row as not-classified). Index #5 is the DB backstop; the app logic must AGREE.
5. **Monitoring** (§A.10) — the signal queries (queue depth, oldest-pending age, oldest-leased age, jobs/drain, dead-letter count, retry count, last-successful-drain, single-flight skips, stale-lease recoveries, kick failures) in a **sibling** dir `scripts/arch-001-card2-sql/` (NOT `scripts/ops/`) + an operator runbook with the alert thresholds (oldest-pending > 5 min, no-successful-drain-within-threshold, dead-letter > 0).
6. **Smoke runbook** (§A.12) — operator-run, post-deploy: canary → sustained/repeated burst (e.g. 3 waves of 5, not one tight burst) → **poll to ACTUAL settle** (pending=0, due retry_scheduled=0, leased=0, no stale leases, last drain after the final submit) → read run-completeness; full A–G; no H/I/J; no duplicate success; every (arg,family,run_mode) cell succeeds or explicit terminal; submit nonblocking; doctrine `evidence_span` clean; relaxed queue SLO (NOT the old 30s direct-dispatch p95). **Plus carry-forward #4: a two-psql-session reclaim-vs-finalize race** (session A holds `finalize_classifier_job` mid-txn; session B runs `reclaim_stale_leases()`; confirm no double-success + consistent terminal row).
7. **Tests** — see below.

OUT (NOT in Card 2): production-wide rollout / percentage-bucket routing — Card 3 (`shouldRouteToQueue` stays smoke-tag-only + DEFAULT-DISABLED). NO Family H. NO detector tiering. NO change to prompts / taxonomy / family keys / schema mirror / Source 6 / audit-lint / `package.json` / the MCP-server provider path. NO redefining the Card-1/2A SQL functions.

## LOCKED carry-forward (Card-2 requirements, from the operator)
1. **Honor `finalize_classifier_job`'s `false` return** = caller lost the lease (reclaimed/expired/wrong owner). The drainer MUST NOT record success, retry-as-if-succeeded, or double-count. A `false` after a real classify → the job is correctly re-claimed + re-run by a later drain (no double-success). Do NOT paper over it.
2. **Size work within the lease.** Lease = 120s; drainer→MCP timeout = 30s. Complete claim→classify→finalize well within 120s so the reclaim-vs-finalize race effectively never fires normally. Do not let per-job work approach the lease.
3. **Active-queue awareness** on enqueue + `findExistingRun` (scope item 4).
4. **Exercise the reclaim-vs-finalize race LIVE** in the smoke runbook (scope item 6).
5. **DB index #5 is the backstop; app logic must agree** with it (no double-dispatch).
6. **Consistent terminal pairs:** `succeeded`↔`status='success'`; `failed_terminal`/`dead_letter`↔`status='failed'` (or `'fallback'` where the existing failure semantics use it). The finalizer trusts the caller for `p_status` (its CHECK constrains only the domain, not the pairing), so the drainer MUST pass a consistent pair.

## LOCKED defaults (operator-confirmed)
- **C = 3** drainer global provider concurrency (Anthropic Tier-1; C ≤ MCP cap).
- **T = 90s** drainer batch wall-clock budget (free Edge plan, 150s ceiling) — exit before the ceiling with margin; a drainer killed at the ceiling must not wedge single-flight (lease TTL self-heals) nor strand claimed jobs (reclaim recovers them).
- **60s pg_cron safety tick** (channel-1 periodic interval; the `cron.schedule` is written-not-applied and operator-scheduled AFTER drainer deploy + Vault seed).
- **`MCP_SERVER_MAX_PROVIDER_CONCURRENCY = 5`/default** backstop (operator restores before live smoke); invariant **C ≤ cap**.
