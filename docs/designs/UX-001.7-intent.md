## Why this brief exists

UX-001.1 through UX-001.6 shipped the professional brand shell, Timeline-first layout, contextual composer, Act / Inspect / Go menu model, annotation primitives, and cross-device QA baseline. UX-001.6 surfaced zero acute fixes across the cross-device matrix (4 hard-blocker + 2 extension viewports), empirically confirming the system is structurally sound.

Its purpose is consolidation: standardize what already shipped into a durable visual design system that future cards reuse, rather than introducing one-off styling. UX-001.7 must NOT discover new UX, introduce new product behavior, or destabilize the cross-device QA result UX-001.6 verified.

The epic-closer status is binding context: UX-001.7's completion report doubles as the UX-001 epic completion report. Every epic-level acceptance criterion from docs/designs/UX-001-epic.md lines 113-163 is verified PASS or marked DEFERRED-TO-UX-001.5A with rationale.

## The central product rule (binding)

UX-001.7 is a consolidation card, not an invention card. The card's success criterion is: future cards (UX-001.5A, UX-001.5B, and any post-epic work) can reference a single, stable design language instead of re-deriving tokens, primitives, or patterns from individual prior cards.

Concretely:
- If a value (color, spacing, typography setting, touch target) appears as a literal in two or more UX-001.{1-6} surfaces, UX-001.7 consolidates it into a token.
- If a visual treatment (chip, badge, ring, outline, focus state) varies across UX-001.{1-6} surfaces, UX-001.7 normalizes it to one canonical treatment.
- If a primitive (UX-001.5's 12 annotation primitives) is consumed inconsistently across surfaces, UX-001.7 audits and aligns the consumption.

If a value appears only once and has no consolidation candidate, UX-001.7 leaves it alone. Do not overbuild.

## The seven workstreams

UX-001.7 ships seven workstreams. The designer's deliverable enumerates which are in v1 scope vs deferred. Workstream 4 (EvidenceAnnotationChip) has an explicit fallback path; the other six are in-scope by default.

**Workstream 1 — Token consolidation.**
Audit all UX-001.{1-6} surfaces for direct style literals and one-off values. Targets: typography, spacing, radius, surface colors, focus rings, touch target sizes, chip styles, badge styles, overlay / scrim styles, keyboard badge styles, annotation colors, composer / context surface styles.

Acceptance: no new arbitrary hex values; no new arbitrary spacing constants where a token exists; no new one-off focus ring styles; no new isolated chip / badge variants; visual primitives reference shared tokens.

**Workstream 2 — Typography and spacing normalization.**
UX-001.1 shipped a minimum typography baseline. Subsequent cards added component-specific text treatments. UX-001.7 consolidates into a small, usable set.

Typography groups (designer finalizes): header / brand, room strip, timeline node, selected context, composer, popout heading, popout body, chip label, badge label, keyboard hint, inspect detail.

Spacing groups (designer finalizes): screen inset, surface gap, compact row gap, chip gap, node internal padding, popout internal padding, composer padding, touch target minimum.

Acceptance: every typography / spacing token has at least two consumers across UX-001.{1-6} surfaces; single-consumer literals stay as literals (do not overbuild).

**Workstream 3 — Annotation primitive integration audit.**
UX-001.5 introduced 12 primitives. UX-001.7 audits consistency of consumption across: node chips, badges, rings, outlines, edge / relationship highlights, Inspect grouped chips, semantic flag chips, referee banner chips, evidence annotations.

Acceptance: a chip looks like a chip everywhere; a passive annotation does not look like a button; a warning / attention surface does not imply truth / verdict; machine / user source hints remain UX-001.5A-future-compatible; legacy semanticFlags string array coexists safely with semanticFlagsChips.

**Workstream 4 — EvidenceAnnotationChip disposition (folded from UX-001.5C).**
Per POSTRUN-UX001 Scope 6 commit e28f765 and docs/designs/UX-001-epic.md line 49, EvidenceAnnotationChip and EvidenceAnnotationStream are folded into UX-001.7's scope.

Decision tree (designer chooses during pre-launch scope-reality audit; operator validates at Stage 2 HALT):

- **Preferred path:** Refactor EvidenceAnnotationChip and EvidenceAnnotationStream to consume the new annotation primitives (AnnotationChip, AnnotationChipStrip, AnnotationBadge, AnnotationFocusRing). M-effort ceiling: if the refactor exceeds 8 hours estimated implementer wall time, OR requires modifying any read-only UX-001.{1-5} file outside the bounded extension envelope, fall back.
- **Fallback path:** Document exactly why the refactor cannot land in UX-001.7 (specific file dependencies, specific test count blowup, specific scope overflow). File UX-001.5C via spawn-card as a focused S-effort separate card with the documented blockers as the design constraints. Leave EvidenceAnnotationChip on its current bespoke implementation.

No untracked ghost deferral is acceptable. The fold is resolved as either refactor-shipped or 5C-filed-with-issue-link.

**Workstream 5 — Cross-device design-system consistency preservation.**
UX-001.6 passed the matrix at 6 viewports. UX-001.7 ensures the consolidation preserves that.

Checks: touch target token consistently used across all surfaces; keyboard hints remain browser-only where intended (the {platformOs, windowWidth} encoding from uxOneOneSixViewportMatrix.test.ts is preserved); mobile avoids keyboard-specific visual clutter; compact room strip remains compact; Timeline offset caps unaffected; composer / context surfaces retain measured layout; Act / Inspect / Go popouts retain platform-appropriate affordances.

Acceptance: UX-001.6's parameterized viewport matrix tests (uxOneOneSixViewportMatrix.test.ts + uxOneOneSixTouchTargets.test.ts + uxOneOneSixColorIndependence.test.tsx + uxOneOneSixReadOnlyBoundary.test.ts) all remain green with zero modification.

**Workstream 6 — Accessibility consistency.**
Consolidate accessibility treatment across: focus labels, accessible chip names, touch target minimums, non-color-only differentiation, keyboard focus rings, popover / scrim semantics, button labels, keyboard hint visibility, Observation / Allegation future-readiness.

UX-001.5A will add label chips. The design system must already be ready to express "Machine observation: Source gap" and "User allegation: Needs source" using only the consolidated tokens, without relying on color (per the cdiscourse-doctrine skill §10a Observations vs Allegations rule).

**Workstream 7 — Final UX-001 design-system handoff documentation + epic completion note.**
UX-001.7 produces TWO documentation surfaces in docs/core/current-status.md:

(a) Phase 7 framing section + final UX-001 design-system handoff. Naming: canonical visual tokens, canonical component primitives, surfaces and where they are used, read-only boundaries for future cards, known intentional divergences, UX-001.5A consumption notes, EvidenceAnnotationChip disposition (refactor-shipped or 5C-filed with link), cross-device QA baseline reference, final UX-001 completion status. This becomes the source of truth for future work.

(b) Separate UX-001 epic completion note. Names UX-001.1 through UX-001.7 as shipped (with merge SHAs and PR numbers), UX-001.5A as next adjacent follow-through (deferred pending source-access audit), UX-001.5B as contingent (filed only if UX-001.5A audit determines classifier sources need backend work). Verifies each epic-level acceptance criterion from UX-001-epic.md L113-163 as PASS or DEFERRED-TO-UX-001.5A with rationale.

1. Workstream 4 audit determines EvidenceAnnotationChip refactor requires modifying a read-only UX-001.{1-5} source file beyond the bounded list — HALT, surface the dependency, designer defaults to FALLBACK path (file UX-001.5C via spawn-card).

5. Test count delta exceeds +1,000 without doctrine-parameterization justification — HALT, surface (the count is a signal something accidentally exploded; UX-001.6's +2,937 was specifically traceable to doctrine matrix parameterization).
6. Any new third-party dependency proposed — HALT immediately.
7. EvidenceAnnotationChip refactor wall-time estimate exceeds 8 hours during implementation — fall back, file UX-001.5C, document.
8. Token count proposed exceeds 50 new tokens — HALT, surface (signal of overbuild; this brief says "do not overbuild").
9. Any doctrine drift (verdict-token leak, internal-code leak, service-role usage) discovered in any UX-001.{1-6} surface during the scope-reality audit — HALT immediately (doctrine fix card, not consolidation).
