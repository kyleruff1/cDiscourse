## 0. Card-vs-reality discrepancies (read first)

Prior designers found several card-vs-code drifts on this roadmap. I verified every model name and field the AN-003 card body implies against the real code. Findings:

### D1 — There is NO `docs/designs/AN-001.md`. AN-001 shipped as code only.

The card says "Compatible with AN-001 deterministic board diagnostics (does not duplicate)." AN-001 exists as a real, merged module — `src/features/analytics/boardDiagnostics.ts` (431 lines, 18 tests in `__tests__/boardDiagnostics.test.ts`) — but there is **no design doc** for it. So "compatible with AN-001" means *compatible with the actual `boardDiagnostics.ts` API*, which this design treats as the source of truth. See §10 for the exact non-duplication boundary.

### D2 — `src/features/analytics/` has NO `index.ts` barrel.

AN-001's `boardDiagnostics.ts` is imported via its full path (`../src/features/analytics/boardDiagnostics`). It is not re-exported through a barrel. AN-003 follows the same convention: a new file at the same path level, imported by full path. This design does **not** create an `index.ts` (out of scope; would be a gratuitous refactor of AN-001's call site).

### D3 — AN-001 operates on a raw argument projection, NOT on the lifecycle map.

The card body for AN-003 implies a "room's argument tree" input and lists metrics phrased in lifecycle terms ("clusters in `open / rebutted / source_requested / quote_requested`"). AN-001 (`boardDiagnostics`) does **not** consume `PointLifecycleMap` at all — it takes a flat `BoardDiagnosticsArgumentInput[]` projection and re-derives standing bands via `inferStandingBand`. AN-003 is the opposite: it is **lifecycle-map-first**. AN-003 consumes the *already-built* `PointLifecycleMap`, `ArgumentTimelineMapModel`, GAME-001 advisory, EV-001 evidence contracts, and the SC-004 dock model — it never re-derives any of them. This is the non-duplication boundary and it is clean: AN-001 = "what does the raw board look like"; AN-003 = "is the lifecycle-mapped tree playable". They share no derivation logic.

### D4 — The card's metric "unresolved points" maps to lifecycle CLUSTER states, not message states.

LIFE-001's `PointLifecycleState` is a per-**cluster** summary (`PointLifecycleClusterSummary.state`), keyed by `branchRootMessageId`. The card's "Number of unresolved points (clusters in `open / rebutted / source_requested / quote_requested`)" and "exhausted / stale points (clusters in `exhausted / moved_on_by_<side> / ignored_by_<side> / ignored_by_both`)" are both **cluster-level counts**. This design counts `PointLifecycleMap.byCluster` entries by `state`. There is no card discrepancy here — just a precision note so the implementer counts clusters, not messages.

### D5 — The 18-state vocabulary the card implies is actually 19 values.

The card references "the 18 PointLifecycleState values". The real `ALL_POINT_LIFECYCLE_STATES` array in `pointLifecycleModel.ts` has **19** entries (it includes `archived_or_resolved`, which the doc comment's "18 values partition…" prose under-counts — the array is the source of truth). AN-003 must iterate `ALL_POINT_LIFECYCLE_STATES` (the exported frozen array) rather than hard-coding a count of 18 or 19. The card's "18" is descriptive shorthand, not a contract. No blocker — just iterate the exported array.

### D6 — "Average actions to reach the active unresolved point from root" needs a defined "active unresolved point".

The card lists this metric without defining "the active unresolved point". `ArgumentTimelineMapModel` has exactly one `activeNode` (and `activePathIds`). This design defines the metric precisely (§4, `actionsToActiveUnresolvedPointFromRoot`): the depth of the timeline's `activeNode` from its cluster root, **only when that node's cluster is in an unresolved state**; `null` otherwise. "Actions" = edges traversed = `node.depth` relative to the cluster root. This is a design decision, surfaced here so the implementer does not have to invent it.

### D7 — "Nodes with no available suggested action" requires the SC-004 dock model, which IS shipped.

The card's red-flag metric "nodes with no available suggested action (the dock would render empty)" maps to `buildTimelineNodeActionDockModel` in `src/features/arguments/timelineNodeActionDockModel.ts` (SC-004, shipped). A node has "no available suggested action" when, for the canonical participant actor, **every** entry in the dock model's `actions[]` is `isDisabled === true`. Note: in practice SC-004 *always* keeps `open_cards_detail` enabled for every actor, and `reply` enabled on non-archived clusters — so a literally-empty dock is near-impossible. The honest, useful metric AN-003 should produce is "nodes whose dock has **no enabled POST-producing action**" (no enabled `reply / challenge / ask_source / ask_quote / clarify / add_evidence / narrow / concede / confirm / branch / synthesize`). See §4 `nodesWithNoActionableSuggestion` for the exact definition and §11 R4 for the risk note. This is the one place the card's literal phrasing ("dock would render empty") is technically unreachable; the design adapts it to a *useful* signal and documents why.

None of D1–D7 is a blocker. AN-003 proceeds with the adaptations documented above.

## Goal

AN-003 is a **pure-model, dev/debug-only** diagnostics deriver. Given a room's already-built lifecycle-mapped argument tree, it answers three questions for an operator tuning GAME-001's advisory thresholds: **is this tree playable, where is it stuck, and what advisory state dominates?**

The motivating constraint (card "Why now"): GAME-001 shipped exhaustion / moved-on advisories with v1 threshold constants (`exhaustionRepeatThreshold: 3`, `movedOnTurnThreshold: 4`, etc.). Without measurement those thresholds get tuned by vibes. AN-003 produces a deterministic `TreePlayabilityDiagnostics` record plus a dev/debug Markdown snapshot so an operator can look at real rooms and see whether the thresholds are firing sensibly.

Doctrine constraints that shape the design (from `cdiscourse-doctrine`, `point-standing-economy`, `evidence-doctrine`, `test-discipline`):

- **No verdict.** AN-003 produces counts and advisory-state distributions. It never labels a room, point, or person as winner / loser / correct / true / false. "Playability" is a structural readout (is the tree stuck?), not a quality judgement.
- **No popularity / engagement input.** AN-003 reads only structural lifecycle / evidence / dock outputs. It never reads heat, like counts, view counts, recent-activity weight, or `standingBand` / `toneBand` / `temperatureBand` from the surface model.
- **Dev-only output channel.** The `TreePlayabilityDiagnostics` object and the Markdown snapshot are operator/QA tooling. They are never imported into `app/`, never rendered to a user, never persisted.
- **No re-derivation.** AN-003 consumes `PointLifecycleMap`, `ArgumentTimelineMapModel`, GAME-001 advisories, EV-001 evidence contracts, and SC-004 dock models. It never re-runs `buildPointLifecycleMap`, never re-derives an axis, never re-classifies a message.
- **Pure TS.** No React, no Supabase, no network, no AI, no `Date.now()` / `Math.random()` in the deriver body. Any time-shaped input is passed in.
- **Compatible with AN-001, no duplication.** AN-001 = raw board shape; AN-003 = lifecycle-mapped playability. Disjoint derivation logic (§10).

AN-003 ships **no migration, no Edge Function, no schema change, no Supabase write, no `.env*` change, no new dependency, no UI, no AI path.**

## Why a separate module (and not an AN-001 extension)?

Three reasons AN-003 is its own file rather than new fields on `boardDiagnostics.ts`:

1. **Different input contract.** AN-001 takes a flat `BoardDiagnosticsArgumentInput[]` projection and re-derives bands itself. AN-003 takes the *built* `PointLifecycleMap` + `ArgumentTimelineMapModel` + GAME-001 advisory + EV-001 contracts + SC-004 dock. Merging the two would force AN-001 callers to build a lifecycle map they do not need, and force AN-003 to re-implement lifecycle reading. Keeping them separate keeps each input contract honest.
2. **Different consumer.** AN-001 is documented (in its own header) as feeding "a future operator-only debug panel" and UI tests. AN-003 explicitly exists to feed **threshold tuning** for GAME-001 and **AN-002 visual-QA snapshots**. Different purpose, different output shape.
3. **Non-duplication is a card acceptance criterion.** "Compatible with AN-001 ... (does not duplicate)." A separate module with a disjoint derivation surface is the cleanest way to satisfy that and to prove it (the test plan includes a structural assertion, §8).

## Data model

### Input — `TreePlayabilityDiagnosticsInput`

AN-003 consumes already-built models. The caller (a dev script or a test) builds them; AN-003 never builds them itself.

```ts
import type { ArgumentTimelineMapModel, ArgumentTimelineMapNode }
  from '../arguments/argumentGameSurfaceModel';
import type { PointLifecycleMap, PointLifecycleState }
  from '../lifecycle';
import type { ExhaustionTimeoutAdvisory } from '../lifecycle';
import type { TimelineEvidenceContract } from '../evidence/evidenceModel';
import type { TimelineNodeActionDockModel } from '../arguments/timelineNodeActionDockModel';

export interface TreePlayabilityDiagnosticsInput {
  /** Already-built timeline surface model (the room's argument tree). */
  timelineMap: ArgumentTimelineMapModel;

  /** Already-built LIFE-001 lifecycle map for the same room. */
  lifecycleMap: PointLifecycleMap;

  /**
   * GAME-001 advisory per cluster id (= branchRootMessageId). Optional;
   * when omitted, the `dominantAdvisoryState` falls back to the LIFE-001
   * cluster state. Keyed by cluster id. Caller builds via
   * `deriveExhaustionTimeoutAdvisory` / the LIFE-001 adapter.
   */
  exhaustionAdvisoryByClusterId?: ReadonlyMap<string, ExhaustionTimeoutAdvisory>;

  /**
   * EV-001 evidence contract lookup. Returns null for a message with no
   * artifacts. Caller supplies the same `getTimelineEvidenceContract`-backed
   * function the room shell uses. Optional — when omitted, evidence-debt
   * concentration is computed from lifecycle source/quote-request states
   * only (a coarser but still correct signal).
   */
  evidenceContractFor?: (messageId: string) => TimelineEvidenceContract | null;

  /**
   * SC-004 dock-model builder for a single node, already bound to the
   * canonical participant actor. AN-003 calls this once per non-root,
   * non-detached node to detect "no actionable suggestion" nodes.
   * Optional — when omitted, `nodesWithNoActionableSuggestion` is reported
   * as `null` (signal unavailable, not zero).
   */
  dockModelForNode?: (messageId: string) => TimelineNodeActionDockModel | null;

  /** Optional config overrides. */
  config?: TreePlayabilityDiagnosticsConfig;
}

export interface TreePlayabilityDiagnosticsConfig {
  /**
   * Branch depth at or above which the tree is flagged "branch overload"
   * (a deep, possibly-stuck chain). Default 8. Advisory only.
   */
  branchOverloadDepthThreshold: number;

  /**
   * Fraction of clusters in an unresolved state at or above which the
   * room is flagged `unresolved_dominant`. Default 0.5. Advisory only.
   */
  unresolvedDominanceFraction: number;

  /**
   * Fraction of clusters in an exhausted/stale state at or above which
   * the room is flagged `stalled`. Default 0.4. Advisory only.
   */
  stalledDominanceFraction: number;
}
```

### Output — `TreePlayabilityDiagnostics`

Every metric the card lists, plus a deterministic fingerprint. All numbers; no prose verdicts.

```ts
/** A playability verdict-FREE structural readout. Dev/debug only. */
export type TreePlayabilityClass =
  | 'empty'              // no messages
  | 'healthy'            // unresolved + stalled both below their thresholds
  | 'unresolved_dominant'// many open/rebutted/requested clusters
  | 'stalled'            // many exhausted/moved-on/ignored clusters
  | 'mixed';             // both dominance flags fire

export interface TreeBranchDepthStats {
  /** Max branch depth across every cluster (deepest path from a cluster
   *  root to a leaf). 0 for a single-message room. */
  maxDepth: number;
  /** Median of per-cluster max depths. Deterministic (lower-mid on ties). */
  medianDepth: number;
  /** True when `maxDepth >= config.branchOverloadDepthThreshold`. */
  isBranchOverload: boolean;
  /** The cluster id of the deepest chain (for the Markdown snapshot). */
  deepestClusterId: string | null;
}

export interface TreeLifecycleStateBreakdown {
  /** Count of clusters in each of the 19 PointLifecycleState values.
   *  Keys cover every entry of ALL_POINT_LIFECYCLE_STATES. */
  countsByState: Readonly<Record<PointLifecycleState, number>>;
  /** Total clusters (= sum of countsByState values). */
  totalClusters: number;
}

export interface TreePlayabilityDiagnostics {
  /** Verdict-free structural classification (see TreePlayabilityClass). */
  playabilityClass: TreePlayabilityClass;

  /** Total messages in the timeline map. */
  totalMessages: number;
  /** Total clusters (= lifecycleMap.byCluster.size). */
  totalClusters: number;

  /** Clusters in open / rebutted / source_requested / quote_requested. */
  unresolvedPointCount: number;
  /** Clusters in exhausted / moved_on_by_* / ignored_by_* / ignored_by_both. */
  exhaustedOrStalePointCount: number;
  /** unresolvedPointCount / totalClusters; 0 when no clusters. */
  unresolvedFraction: number;
  /** exhaustedOrStalePointCount / totalClusters; 0 when no clusters. */
  exhaustedOrStaleFraction: number;

  /** Branch depth statistics (max + median + overload flag). */
  branchDepth: TreeBranchDepthStats;

  /**
   * Average actions (edges) to reach the active unresolved point from its
   * cluster root. null when the timeline has no activeNode OR the active
   * node's cluster is not in an unresolved state. See §0/D6 + §4.
   */
  actionsToActiveUnresolvedPointFromRoot: number | null;

  /**
   * Source / evidence debt concentration: fraction of clusters that carry
   * unresolved evidence debt (open source/quote request, OR a worst EV-001
   * status of no_source / broken on a claim-shape member). [0, 1].
   */
  evidenceDebtConcentration: number;
  /** Absolute count behind `evidenceDebtConcentration`. */
  evidenceDebtClusterCount: number;

  /**
   * Count of non-root, non-detached nodes whose SC-004 dock has no enabled
   * POST-producing action (the "dock would render empty" red flag, adapted
   * per §0/D7). null when `dockModelForNode` was not supplied.
   */
  nodesWithNoActionableSuggestion: number | null;

  /** Full per-state cluster breakdown (all 19 lifecycle states). */
  lifecycleBreakdown: TreeLifecycleStateBreakdown;

  /**
   * The advisory state that dominates the room: the most-frequent GAME-001
   * advisory state across clusters (ties broken by ExhaustionTimeoutAdvisory
   * priority order), or null when no cluster has a GAME-001 advisory.
   */
  dominantAdvisoryState: PointLifecycleState | null;
  /** Count of clusters carrying `dominantAdvisoryState`. 0 when null. */
  dominantAdvisoryClusterCount: number;

  /** Stable count-based fingerprint (FNV-1a, no PII). Change-detection. */
  fingerprint: string;
}
```

## API / interface contracts

The implementer (and any caller) needs these exact contracts:

1. **`computeTreePlayabilityDiagnostics(input)`** — see signature above. Pure, total, deterministic. Returns a fully-populated `TreePlayabilityDiagnostics`. For an empty timeline (`timelineMap.nodes.length === 0`) it returns the canonical empty record: `playabilityClass: 'empty'`, all counts 0, `branchDepth` `{ maxDepth: 0, medianDepth: 0, isBranchOverload: false, deepestClusterId: null }`, `actionsToActiveUnresolvedPointFromRoot: null`, `dominantAdvisoryState: null`, `fingerprint` of the empty input.

2. **`renderTreePlayabilityMarkdown(diagnostics, options?)`** — pure string. Produces the dev/debug Markdown body (§5). Never writes a file.

3. **Lifecycle-state classification sets** (internal constants, exported via `_internal` for tests):
   ```ts
   const UNRESOLVED_STATES: ReadonlySet<PointLifecycleState> = new Set([
     'open', 'rebutted', 'source_requested', 'quote_requested',
   ]);
   const EXHAUSTED_OR_STALE_STATES: ReadonlySet<PointLifecycleState> = new Set([
     'exhausted',
     'moved_on_by_affirmative', 'moved_on_by_negative',
     'ignored_by_affirmative', 'ignored_by_negative', 'ignored_by_both',
   ]);
   ```
   These exactly mirror the card body's two parenthesised lists. The implementer must NOT add `branch_recommended` to either set — it is neither "unresolved" nor "stale"; it is a routing advisory. It is still counted in `lifecycleBreakdown.countsByState`.

4. **Branch-depth contract.** For each cluster, AN-003 reads `PointLifecycleMap.byCluster` to get cluster ids, then for each cluster computes the max `node.depth` *relative to the cluster root* across that cluster's member nodes from the timeline map. `node.depth` is the timeline map's absolute depth field; cluster-relative depth = `node.depth - clusterRootNode.depth`. `medianDepth` is the median of those per-cluster max depths (lower-middle element on an even count, for determinism). The implementer must use `ArgumentTimelineMapNode.depth` and `branchRootMessageId` — not recompute the tree.

5. **`dockModelForNode`** — the caller passes a function already bound to the *canonical participant actor* (`'other'`). AN-003 must not pick the actor itself; the diagnostic is "is there any actionable suggestion for a regular participant on this node". The implementer iterates the dock model's `actions[]` and checks whether at least one POST-producing action (`reply / challenge / ask_source / ask_quote / clarify / add_evidence / narrow / concede / confirm / branch / synthesize`) has `isDisabled === false`.

6. **`fingerprint`** — FNV-1a 32-bit over a `:`-joined count tuple (same helper shape as `boardDiagnostics.fnv1a`). Tuple includes: `totalMessages`, `totalClusters`, `unresolvedPointCount`, `exhaustedOrStalePointCount`, `branchDepth.maxDepth`, `branchDepth.medianDepth`, `evidenceDebtClusterCount`, `nodesWithNoActionableSuggestion ?? -1`, and the count of each lifecycle state in `ALL_POINT_LIFECYCLE_STATES` order. The implementer copies AN-001's `fnv1a` rather than importing it (do not create a cross-module coupling for one helper; or, acceptably, lift `fnv1a` into a tiny shared `src/lib/fnv1a.ts` — implementer's call, but the simplest non-coupling choice is a local copy with a comment pointing at AN-001).

## Edge cases

The implementer must handle each:

- **Empty timeline** (`nodes.length === 0`). Return the canonical empty record (§API #1). `lifecycleMap.byCluster.size` will also be 0. No division by zero — guard every fraction with `totalClusters === 0 ? 0 : …`.
- **Single-message room** (root only, no replies). One cluster, state `open`. `branchDepth.maxDepth = 0`, `medianDepth = 0`. `unresolvedPointCount = 1`. `actionsToActiveUnresolvedPointFromRoot` = 0 if the root is the active node and its cluster is unresolved, else `null`.
- **No `activeNode`.** `ArgumentTimelineMapModel.activeNode` can be `null`. Then `actionsToActiveUnresolvedPointFromRoot = null` (not 0).
- **Active node's cluster is resolved** (e.g. `conceded` / `synthesis_ready` / `archived_or_resolved`). The metric is `null` — there is no *active unresolved point* to measure.
- **`exhaustionAdvisoryByClusterId` omitted.** `dominantAdvisoryState` falls back to: the most-frequent LIFE-001 cluster `state` among states in `EXHAUSTED_OR_STALE_STATES` (so the operator still sees the dominant *advisory* state even without the GAME-001 input). If no cluster is in an exhausted/stale state, `dominantAdvisoryState = null`.
- **`evidenceContractFor` omitted.** `evidenceDebtConcentration` is computed from lifecycle cluster states alone — a cluster counts as carrying debt when its `hasOpenSourceOrQuoteRequest` is true OR its `state` is `source_requested` / `quote_requested`. Coarser, but never wrong (no false "anchored" claims).
- **`dockModelForNode` omitted.** `nodesWithNoActionableSuggestion = null` (signal unavailable — never silently report 0, which would falsely imply "every node is actionable").
- **Detached nodes** (`node.isDetached === true`). Excluded from the `nodesWithNoActionableSuggestion` scan (a detached node legitimately has no tree-attached action; counting it would inflate the red-flag metric). Detached nodes still count toward `totalMessages`.
- **Cluster with a root not present in the timeline map** (data skew). Skip the cluster for branch-depth (treat as depth 0); still count its lifecycle state. Never throw.
- **Cycle-shaped input.** `node.depth` is precomputed by the timeline map builder, which is already cycle-defensive; AN-003 reads it and never walks parent chains itself, so AN-003 cannot loop. (This is a deliberate design choice — read precomputed `depth`, do not re-walk.)
- **Concurrent edits / offline / network failure.** Not applicable — AN-003 is a pure function over in-memory models. No network, no I/O.
- **Permission-denied.** Not applicable — pure model, no Supabase, no RLS surface.
- **Doctrine edge case — "what if a viral cluster has 0 evidence but high engagement?"** AN-003 reads no engagement signal at all. A wrong-but-loud cluster and a right-but-quiet cluster with identical move structure produce identical diagnostics. The `evidenceDebtConcentration` metric counts the *absence of evidence*; it never counts popularity as a substitute for evidence (evidence-doctrine §"What does NOT count as evidence").
- **Doctrine edge case — "does heat raise the playability class?"** No. `playabilityClass` is derived purely from `unresolvedFraction` and `exhaustedOrStaleFraction` vs config thresholds. Heat / temperature bands are never read.

## Dev / debug Markdown output

The card requires: "Dev / debug Markdown output for AN-002 visual QA snapshots ... Dev / debug output only. Never user-facing."

### How it is produced

- **`renderTreePlayabilityMarkdown(diagnostics, options)`** (in `treePlayabilityDiagnostics.ts`) is the pure string builder. It is unit-tested directly. It never touches the filesystem.
- **`scripts/diagnostics/buildTreePlayabilitySnapshot.js`** is the operator-invoked dev script. It:
  1. Loads a room fixture (a JSON file of `ArgumentMessageInput[]` — the same shape `buildArgumentTimelineMap` already accepts) from a `--fixture <path>` flag, or a built-in deep-tree fixture in `--fixture-builtin` / dry mode.
  2. Builds `ArgumentTimelineMapModel` via `buildArgumentTimelineMap`, `PointLifecycleMap` via `buildPointLifecycleMap`, GAME-001 advisories via the LIFE-001 adapter, and EV-001 contracts via `getTimelineEvidenceContract` — exactly the same builders the room shell uses.
  3. Calls `computeTreePlayabilityDiagnostics` then `renderTreePlayabilityMarkdown`.
  4. Writes the Markdown to **`artifacts/diagnostics/tree-playability/<timestamp>-<roomLabel>-tree-playability.md`** (the `artifacts/` tree is gitignored — confirm `artifacts/` is in `.gitignore`; `docs/diagnostics/diagnostic-inspect-package.md` already uses `artifacts/diagnostics/`, so it is). `--dry` prints to stdout and writes nothing.

### Why `artifacts/`, not `docs/`

`docs/testing-runs/` holds *committable* run reports. AN-003 snapshots are throwaway QA artifacts for threshold tuning — they go to the gitignored `artifacts/diagnostics/` tree alongside the existing diagnostic-inspect-package output. This keeps the repo clean and makes it impossible for a snapshot to drift into a user-facing surface.

### Markdown body shape (produced by `renderTreePlayabilityMarkdown`)

A single Markdown document, no verdict copy, no internal codes leaked as prose:

```
# Tree playability snapshot — <roomLabel>

Generated: <generatedAtLabel>   (dev/debug only — not a user-facing report)

## Summary
- Playability class: <playabilityClass>
- Messages: <n>   Clusters: <n>

## Resolution
- Unresolved points: <n> (<pct>%)
- Exhausted / stale points: <n> (<pct>%)
- Evidence-debt concentration: <pct>% (<n> clusters)

## Branch depth
- Max depth: <n>   Median depth: <n>
- Branch overload: <yes|no>

## Reachability
- Actions to active unresolved point from root: <n | n/a>
- Nodes with no actionable suggestion: <n | signal unavailable>

## Advisory distribution
- Dominant advisory state: <state | none> (<n> clusters)
- Per-state cluster counts:
  | Lifecycle state | Clusters |
  | --- | --- |
  | open | <n> |
  ... (all 19 states) ...

## Fingerprint
<fnv1a hex>
```

`<playabilityClass>` and the lifecycle-state names are *internal codes shown deliberately* — this is a dev/debug surface for an operator, not a user-facing string, so the `gameCopy.toPlainLanguage` mapping is intentionally NOT applied (and the doc header says so explicitly). The ban-list test (§8) still scans the output to guarantee no *verdict* token (`winner`, `true`, `loser`, …) ever appears — internal lifecycle codes are allowed, verdict words are not.

## Risks

- **R1 — `node.depth` semantics.** AN-003's branch-depth math assumes `ArgumentTimelineMapNode.depth` is the absolute depth from the timeline root, and that cluster-relative depth = `node.depth - clusterRootNode.depth`. The implementer must confirm this against `buildArgumentTimelineMap` before relying on it; if `depth` is already cluster-relative, drop the subtraction. A one-line fixture test (test #8) catches a wrong assumption.
- **R2 — SC-004 dock is rarely literally empty.** Per D7, `buildTimelineNodeActionDockModel` keeps `open_cards_detail` (and usually `reply`) enabled for every actor. The card's "dock would render empty" is technically unreachable. AN-003 adapts the metric to "no enabled POST-producing action", which IS reachable (archived/resolved clusters, detached nodes, observer-only contexts). The implementer must use the adapted definition; reporting literal-empty would always be 0 and the metric would be useless.
- **R3 — GAME-001 advisory keying.** `exhaustionAdvisoryByClusterId` must be keyed by `branchRootMessageId` (cluster id), matching `PointLifecycleMap.byCluster`. If a caller keys by message id, `dominantAdvisoryState` silently under-counts. Document the key contract in the JSDoc and assert it in a fixture test.
- **R4 — Median definition drift.** "Median depth" must be deterministic. The design fixes it to the lower-middle element on an even count. The implementer must not use an averaging median (would yield non-integer depths and a fragile fingerprint). Test #8 pins it.
- **R5 — Existing tests.** No existing test should need updating — AN-003 is purely additive (new file, new suite). If `npm run test` shows a changed count anywhere other than the new suite, something was modified that should not have been.
- **R6 — `artifacts/` gitignore.** The dev script writes to `artifacts/diagnostics/tree-playability/`. Confirm `artifacts/` is gitignored before committing the script (it is — the diagnostic-inspect-package already uses `artifacts/diagnostics/`). A snapshot must never be committed.

- *No truth labels.* `TreePlayabilityClass` values are `empty / healthy / unresolved_dominant / stalled / mixed` — structural states, not verdicts. `_forbiddenTreePlayabilityTokens()` + test #24 scan every produced string for `winner / loser / correct / true / false / liar / right / wrong / proven`. **`point-standing-economy`:**
- *Bands are gameplay analysis, not verdicts.* AN-003 does not read or emit `standingBand` at all — it reads only lifecycle / evidence / dock structure. It cannot mislabel a band as a verdict because it never touches bands. PASS.

- *Debt is "lacks X", never "is false".* `evidenceDebtConcentration` and `evidenceDebtClusterCount` are framed as "lacks evidence" counts; no metric implies a debt-carrying claim is false. PASS.
- *Engagement credit vs factual-standing credit stay separate.* AN-003 reports neither — it reports structural debt only. PASS.
