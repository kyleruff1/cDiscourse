## Goal

Make the composer feel like one persistent, contextual command center
across all 14 canonical modes, four required viewports, and both touch +
keyboard input — without changing the submit path, without modifying
UX-001.2's repositioned readout / score tracker / strip placements, and
without pushing the Timeline back above UX-001.2's hard offset caps
(128 phone / 168 tablet / 200 wide). The composer's existing one-box
chassis (QOL-030 `OneBox` + `boxModel` per-type draft buffers + `ActPopout`
flash menu) is the foundation; UX-001.3 closes the consolidation gap by
(a) introducing a **collapsed** composer state that persists below the
Timeline so the user always sees what they are acting on, (b) extending
draft persistence from per-mode-only to per-mode × per-target,
(c) consolidating validation into compact contextual signals attached to
field or composer header (no full-screen modals, no toast banners, no
verdict copy), and (d) wiring focus-context routing so composer keyboard
shortcuts and board keyboard shortcuts never collide.

Doctrine anchors: `cdiscourse-doctrine` §1 (no verdict labels), §4 (no AI
calls from the production app, no AI authoritative flags), §9 (no internal
codes leak into user-facing strings); `accessibility-targets` (44×44 hit
targets, role+label+state on every Pressable, color-independent
affordances, reduce-motion); `expo-rn-patterns` (no new dep speculatively,
RN primitives first, mode chip routes through existing tokens).

## §4 — Mode-switching implementation (Q4)

**Mode switcher placement**: stays in the existing OneBox header type chip (`OneBox.tsx:248-262`). The chip is a 44px-min Pressable that opens the `ActPopout` flash menu. UX-001.3 does NOT introduce a parallel mode switcher (a second segmented control would push the composer header above 72 px and violate the brief's "switcher must be reachable in compact form without consuming significant composer real estate"). New pure-TS module `src/features/arguments/composer/composerDraftRegistry.ts`:

```ts
// Pure TS — no React, no Supabase, no network.
import type { BoxType, Draft, DraftBuffers } from '../oneBox/boxModel';
import { ALL_BOX_TYPES, EMPTY_DRAFT, createEmptyDraftBuffers, isDraftEmpty } from '../oneBox/boxModel';

/** Keyed `(targetKey, BoxType) → Draft`. */
export type ComposerDraftRegistry = Readonly<Record<string, DraftBuffers>>;

/** Root-claim sentinel; all other targets key on `argumentRow.id`. */
export const ROOT_TARGET_KEY = '__root__';

export function deriveTargetKey(parentId: string | null): string {
  return parentId ?? ROOT_TARGET_KEY;
}

export function createEmptyComposerDraftRegistry(): ComposerDraftRegistry {
  return Object.freeze({});  // lazy: buffers materialize on first read
}

export function getDraft(
  registry: ComposerDraftRegistry,
  targetKey: string,
  boxType: BoxType,
): Draft {
  const buffers = registry[targetKey];
  if (!buffers) return EMPTY_DRAFT;
  return buffers[boxType] ?? EMPTY_DRAFT;
}

export function setDraft(
  registry: ComposerDraftRegistry,
  targetKey: string,
  boxType: BoxType,
  draft: Draft,
): ComposerDraftRegistry {
  const existing = registry[targetKey] ?? createEmptyDraftBuffers();
  const next: Record<BoxType, Draft> = { ...existing };
  next[boxType] = draft;
  return Object.freeze({ ...registry, [targetKey]: Object.freeze(next) });
}

export function clearDraft(
  registry: ComposerDraftRegistry,
  targetKey: string,
  boxType: BoxType,
): ComposerDraftRegistry {
  if (!registry[targetKey]) return registry;
  return setDraft(registry, targetKey, boxType, EMPTY_DRAFT);
}

/** True iff EVERY buffer in the registry is empty. */
export function isRegistryEmpty(registry: ComposerDraftRegistry): boolean {
  for (const key of Object.keys(registry)) {
    const buffers = registry[key];
    for (const t of ALL_BOX_TYPES) {
      if (!isDraftEmpty(buffers[t])) return false;
    }
  }
  return true;
}
```

New hook `src/features/arguments/composer/useComposerDraftRegistry.ts`:

```ts
import { useCallback, useRef, useState } from 'react';
import type { BoxType, Draft } from '../oneBox/boxModel';
import {
  ComposerDraftRegistry,
  createEmptyComposerDraftRegistry,
  deriveTargetKey,
  getDraft,
  setDraft,
} from './composerDraftRegistry';

export interface UseComposerDraftRegistryResult {
  /** Read the active draft for (targetKey, boxType). Returns EMPTY_DRAFT if none. */
  readDraft: (targetKey: string, boxType: BoxType) => Draft;
  /** Write a draft for (targetKey, boxType). Pure registry update, no AsyncStorage. */
  writeDraft: (targetKey: string, boxType: BoxType, draft: Draft) => void;
  /** True iff a (targetKey, boxType) buffer currently holds non-empty content. */
  hasDraft: (targetKey: string, boxType: BoxType) => boolean;
  /** Snapshot the entire registry — used by tests + the parked-buffer indicator. */
  snapshot: ComposerDraftRegistry;
}

export function useComposerDraftRegistry(debateId: string): UseComposerDraftRegistryResult {
  const [registry, setRegistry] = useState<ComposerDraftRegistry>(createEmptyComposerDraftRegistry);
  // debateId reset: when the user navigates between rooms, the registry zeroes.
  const lastDebateIdRef = useRef(debateId);
  if (lastDebateIdRef.current !== debateId) {
    lastDebateIdRef.current = debateId;
    setRegistry(createEmptyComposerDraftRegistry());
  }
  const readDraft = useCallback((targetKey: string, boxType: BoxType) =>
    getDraft(registry, targetKey, boxType), [registry]);
  const writeDraft = useCallback((targetKey: string, boxType: BoxType, draft: Draft) =>
    setRegistry((r) => setDraft(r, targetKey, boxType, draft)), []);
  const hasDraft = useCallback((targetKey: string, boxType: BoxType) =>
    !isDraftEmpty(getDraft(registry, targetKey, boxType)), [registry]);
  return { readDraft, writeDraft, hasDraft, snapshot: registry };
}
```

**Wiring into `OneBox.tsx`**: the OneBox is the host. It threads `useComposerDraftRegistry(debate.id)` and computes `targetKey = deriveTargetKey(parentArgument?.id ?? null)`. On `handleSelectBoxType`, it (a) snapshots the CURRENT (targetKey, prevBoxType) body into the registry from the underlying `ArgumentComposer` draft, (b) updates `boxState.type` via `switchBoxType`, (c) reads the NEW (targetKey, newBoxType) body from the registry, (d) hydrates the underlying `ArgumentComposer`'s draft via a new prop `initialBody?: string | null` (additive optional — defaults to null = no override). When the new mode's registry entry is empty, the composer renders an empty body.

**The single-slot `useArgumentComposer` hook remains**, but the OneBox-mounted ArgumentComposer no longer trusts it as the source of truth for "what mode am I in." The `ArgumentComposer`'s `draft.argumentType` / `draft.body` etc. continue to read/write `state.snapshot.activeDraft` via `useArgumentComposer` (AsyncStorage path preserved for cross-session recovery in the worst-case scenario), but the registry's writes happen IN ADDITION — on every `boxType` switch, the OneBox parks the body into the registry under (oldTargetKey, oldBoxType) BEFORE re-typing.

Switch latency: `setRegistry` and `switchBoxType` both happen in the same React batched update; the underlying `ArgumentComposer` receives the new `initialBody` patch on the next render. This is "within one user interaction" per the brief.

Lifecycle: drafts persist across mode switches within the OneBox session. When the dock closes (Cancel) or the user posts a move successfully, the registry is unchanged — drafts for OTHER modes survive. When the user navigates to a different debate, the registry is reset (via the `lastDebateIdRef` guard above). The brief explicitly defers cross-app-session persistence.

New pure-TS module `src/features/arguments/composer/composerKeyboardModel.ts`:

export type ComposerKeyEffect =
  | { type: 'none' }
  | { type: 'submit' }
  | { type: 'open_mode_switcher' }
  | { type: 'close' }
  | { type: 'tab_next' }
  | { type: 'tab_prev' };

export interface ComposerKeyInput {
  key: string;
  metaKey: boolean;
  ctrlKey: boolean;
  shiftKey: boolean;
  /** True iff focus is inside the composer (set by useComposerFocusContext). */
  composerFocused: boolean;
}

export function resolveComposerKeyEffect(input: ComposerKeyInput): ComposerKeyEffect {
  if (!input.composerFocused) return { type: 'none' };
  const cmdOrCtrl = input.metaKey || input.ctrlKey;
  if (cmdOrCtrl && input.key === 'Enter') return { type: 'submit' };
  if (cmdOrCtrl && (input.key === 'k' || input.key === 'K')) return { type: 'open_mode_switcher' };
  if (input.key === 'Escape') return { type: 'close' };
  // Tab/Shift+Tab — let the DOM handle naturally; we return 'none' to avoid
  // prevent-default (the brief specifies Tab moves between FIELDS — that is
  // the browser default; we do not need to interfere).
  return { type: 'none' };
}

UX-001.4 may re-group. |
| Semantic advisory chips overlap with required field validation | **Precedence rule**: header chip status is the worst currently active condition (danger > warning > ready).
