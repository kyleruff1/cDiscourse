## Goal (one paragraph)

MCP-016 implements the server-side boundary designed in MCP-009 — the `semantic-referee` Edge Function — **built entirely on the deterministic `mock` provider**, so the structural seam between the deterministic CDiscourse stack and a (future) AI provider exists, is JWT-protected, is schema-validated, and is fully tested, **without any live AI in the loop**. It is the third structural card in the MCP readiness sequence (roadmap §5: pure-TS foundation → pure-TS consumers → this server boundary → presentation), built deliberately after MCP-011's adversarially-tested validator so the boundary's outbound schema can be proven equivalent to the Node-side contract by a parity test (roadmap §5.1). Four doctrine constraints shape the design and are blockers if violated: (1) **mock-by-default** — the provider registry defaults to `mock`; turning the feature flag on never starts a live call; the `anthropic` / `mcp` slots are stubbed `{ enabled: false }` and physically cannot run (cdiscourse-doctrine §7, roadmap §7); (2) **disabled-by-default is silent** — `SEMANTIC_REFEREE_ENABLED` unset → `{ enabled: false }`, HTTP 200, no provider selected, no key read (MCP-009 §"Disabled-by-default behavior"); (3) **no client-side AI, no privileged write** — the call lives only in the Edge Function, the function builds only a caller-scoped (RLS) client, performs no service-role write, and never inserts into `public.arguments` (cdiscourse-doctrine §4, supabase-edge-contract); (4) **the packet is structural-only** — `authoritative` is the literal `false`, there is no `block` field, no truth field; the outbound Zod schema rejects any attempt to widen the contract (MCP-001 §7, cdiscourse-doctrine §1). MCP-016 owns `supabase/functions/semantic-referee/*`, `supabase/functions/_shared/semanticReferee/*` (mock + fixture providers only), the `classifyMove` client wrapper in `src/lib/edgeFunctions.ts`, and the `[functions.semantic-referee]` block in `supabase/config.toml` (roadmap §6). It is implementation-ready: every dependency is merged.

## Data model

**No new database table. No migration. No new persisted entity. No RLS policy.** This card adds zero rows to any schema and writes nothing.

The boundary transports two shapes only:

1. **Inbound** — `ClassifyMoveRequest`, a transient Edge Function request body, never persisted (defined in §"API / interface contracts").
2. **Outbound** — `ClassifyMoveOutcome`, a transient response body: the discriminated union `{ enabled: false; reason } | { enabled: true; packet: SemanticRefereePacket }`. The `SemanticRefereePacket` is MCP-001's locked contract, mirrored on the Deno side (§"Node ↔ Deno types reconciliation").

The function reads room/parent context through the **caller-scoped client** (RLS-enforced) exactly as `process-language-draft` reads the `debates` table today, and **writes nothing** — no `public.arguments` row, no `flags` row, no audit table, no cache table. The MCP-004 v0 cache is in-memory (owned by MCP-012's `semanticCache.ts`); the boundary does not persist a packet. If a persisted packet cache is ever wanted, that is a future MCP-004 migration in a future card, never this one.

## API / interface contracts

### Inbound request — `ClassifyMoveRequest`

Transports MCP-009's `ClassifyMoveRequest` verbatim. Bodies arrive **already redacted** by the caller; the function runs a defensive second pass (`redaction.ts`) before any provider sees them.

```ts
// supabase/functions/_shared/semanticReferee/types.ts  +  src/lib/edgeFunctions.ts
export interface ClassifyMoveRoomContext {
  debateMode?: string;        // GAME-003 room mode code
  selectedAction?: string;    // quick-action preset
  selectedMoveType?: string;  // argumentType
  side?: 'affirmative' | 'negative' | 'observer' | 'moderator';
  actorRole?: 'initiator' | 'primary_opponent' | 'chime_in' | 'observer';
}

export interface ClassifyMoveRequest {
  roomId: string;                       // RLS-checked: caller must be able to see this room
  moveId?: string;                      // omitted for a pre-send draft
  parentId?: string;
  moveBodyRedacted: string;             // 1..8000 chars (matches process-language-draft cap)
  parentBodyRedacted?: string;          // <= 8000 chars
  roomContext: ClassifyMoveRoomContext;
  requestedClassifiers: string[];       // 1..5 entries, each a SemanticClassifierId
  promptVersionHint?: string;
  contentHash: string;                  // hash of moveBodyRedacted — non-empty
}
```

The request carries **no `block` field, no truth field, no score delta** — there is nothing in the inbound shape that can ask the model for a verdict.

`ClassifyMoveRequestSchema` (`schema.ts`, `npm:zod@4`, `.strict()`):

- `roomId`, `contentHash` — non-empty bounded strings.
- `moveBodyRedacted` — `z.string().min(1).max(8000)`. Empty → `validation_failed`.
- `parentBodyRedacted` — optional `z.string().max(8000)`.
- `moveId`, `parentId`, `promptVersionHint` — optional bounded strings.
- `roomContext` — `.strict()` object; `side` / `actorRole` enums; `debateMode` / `selectedAction` / `selectedMoveType` optional bounded strings.
- `requestedClassifiers` — `z.array(z.enum(ALL_SEMANTIC_CLASSIFIER_IDS)).min(1).max(5)`. Out-of-range or unknown id → `validation_failed`. (MCP-001 §9 caps a packet prompt at 5.)
- Any unknown top-level key → `validation_failed` (the schema is `.strict()`).

### Outbound response — `ClassifyMoveOutcome`

```ts
export type ClassifyMoveOutcome =
  | { enabled: false; reason: 'disabled' | 'not_configured' | 'not_implemented' }
  | { enabled: true; packet: SemanticRefereePacket };
```

`SemanticRefereePacket` is MCP-001's locked shape (mirrored from MCP-011 §7). The boundary adds, removes, renames **no** field. Invariants the outbound `SemanticRefereePacketSchema` enforces:

- `authoritative` — `z.literal(false)`. A non-`false` or absent value fails validation → deterministic fallback packet.
- `packetVersion` — `z.literal('mcp-semantic-referee-v0')`.
- `provider` — `z.enum(['mock', 'mcp', 'anthropic', 'operator-selected'])`. In this card the registry only ever sets `'mock'`.
- Every `binaries[].value` — `z.union([z.literal(0), z.literal(1)])`.
- Every `binaries[].classifierId` ∈ `ALL_SEMANTIC_CLASSIFIER_IDS`.
- `scoreHints` — `.strict()` object of the six MCP-001 integer fields, each `z.int().min(0).max(3)`.
- `.strict()` everywhere — any unknown field (a smuggled `block`, a truth field, a `winner`) fails validation. The boundary **cannot, by construction, return something that hard-gates a post.**

Note: this card's `reason` union omits MCP-009's `'key_missing'` — there is no live provider in MCP-016, so a key is never read and `key_missing` is never produced. `'key_missing'` re-enters the union only when the live-provider pilot card adds the `anthropic` provider. `'not_implemented'` covers the stubbed `anthropic`/`mcp` slots; `'not_configured'` covers an unknown provider value.

### Edge Function shape — `semantic-referee/index.ts`

Follows the `supabase-edge-contract` standard and copies `process-language-draft/index.ts`:

1. **CORS preflight** → `if (req.method === 'OPTIONS') return new Response('ok', { headers: corsHeaders });`
2. **Method guard** → non-`POST` → `methodNotAllowed()` (HTTP 405).
3. **Auth** → no `Authorization` header → `unauthorized()` (HTTP 401), before any body parse. Then `callerClient.auth.getUser()`; `userError || !user` → `unauthorized()`.
4. **Parse body** → invalid JSON → `badRequest('Invalid JSON body.')` (HTTP 400).
5. **Validate** → `ClassifyMoveRequestSchema.safeParse`; on failure → `validationFailed({ error: 'validation_failed', issues })` (HTTP 422).
6. **Room-access RLS check** → `callerClient.from('debates').select('id').eq('id', input.roomId).maybeSingle()`. No row → `validationFailed({ error: 'room_not_found_or_not_accessible' })`. The caller-scoped client means RLS decides visibility. **No service-role client is built** — the function performs no privileged write (supabase-edge-contract: "Build service-role client ONLY if a privileged write is required").
7. **Defensive redaction** → run `redactClassifyMoveRequest(input)` over `moveBodyRedacted` / `parentBodyRedacted` before delegating.
8. **Delegate** → `classifyWithConfiguredProvider(redactedInput)`.
9. **Return** → `ok(outcome)` — `{ enabled: false, reason }` **or** `{ enabled: true, packet }`, both **HTTP 200**. Disabled is a normal outcome, not an error.

The function **never** calls `submit-argument`, **never** inserts into `public.arguments`, **never** inserts a `flags` row, **never** writes any table, **never** builds a service-role client. It is read-context + classify + return.

### Provider registry shape — `classifyWithConfiguredProvider`

```ts
// supabase/functions/_shared/semanticReferee/providers.ts — mock-only build.
export async function classifyWithConfiguredProvider(
  request: ClassifyMoveRequest,
): Promise<ClassifyMoveOutcome> {
  // 1. Disabled-by-default — checked BEFORE any provider is selected.
  const enabled = Deno.env.get('SEMANTIC_REFEREE_ENABLED');
  if (enabled !== 'true') {
    return { enabled: false, reason: 'disabled' };
  }

  // 2. Provider selection — DEFAULT IS 'mock' (doctrine deviation from
  //    languageProcessing/providers.ts, which defaults to 'anthropic').
  const providerName = Deno.env.get('SEMANTIC_REFEREE_PROVIDER') ?? 'mock';

  // 3. Only mock + fixture are WIRED. anthropic + mcp are STUBBED OFF.
  if (providerName === 'mock') {
    return { enabled: true, packet: runMockClassifier(request) };
  }
  if (providerName === 'fixture') {
    return { enabled: true, packet: runFixtureClassifier(request) };
  }
  if (providerName === 'anthropic' || providerName === 'mcp') {
    // STUB — present in the registry, never callable in this card.
    // The live provider is the separate Phase-E operator-approved pilot card.
    return { enabled: false, reason: 'not_implemented' };
  }
  return { enabled: false, reason: 'not_configured' };
}
```

The registry's provider table, conceptually:

| Slot | Status in MCP-016 | Behavior |
|---|---|---|
| `mock` | **wired (default)** | Deterministic `runMockClassifier` — no network, no key. Returns a schema-valid `provider: 'mock'` packet. |
| `fixture` | **wired** | `runFixtureClassifier` — checked-in fixture map keyed by `contentHash`, falls through to `mock` on a miss. No network, no key. |
| `anthropic` | **stubbed off** | `{ enabled: false, reason: 'not_implemented' }`. No `anthropicProvider.ts` file exists. Not callable. |
| `mcp` | **stubbed off** | `{ enabled: false, reason: 'not_implemented' }`. No `mcpAdapter.ts` file exists. Not callable. |
| (anything else) | n/a | `{ enabled: false, reason: 'not_configured' }`. |

**The `mock` default is load-bearing and doctrine-driven.** `languageProcessing/providers.ts` defaults its provider to `'anthropic'`; the semantic referee **must** default to `'mock'` (MCP-001 §10, cdiscourse-doctrine §7, MCP-009 §"Routing rules", roadmap §7). An implementer copying the language-processing registry verbatim would introduce a doctrine violation. The `?? 'mock'` fallback has its own dedicated test (`semanticProviderRegistry.test.ts`).

### Disabled-by-default behavior

The whole layer is **off unless the operator explicitly turns it on**:

1. `SEMANTIC_REFEREE_ENABLED` unset, empty, or any value other than the exact string `'true'` → registry returns `{ enabled: false, reason: 'disabled' }` **before any provider is selected**.
2. The Edge Function returns that object with **HTTP 200** — no 4xx, no 5xx, no stack trace.
3. `classifyMove()` returns `{ ok: true, data: { enabled: false, reason: 'disabled' } }`.
4. The consuming feature (MCP-013 ledger, MCP-014 banners) sees `enabled: false` and **falls back to the deterministic layer-1 result with no error surfaced to the user.** The game still works.
5. `SEMANTIC_REFEREE_ENABLED=true` **but** `SEMANTIC_REFEREE_PROVIDER` unset → provider is `mock` — turning the flag on **never** silently starts a live call. The deterministic mock answers.
6. `SEMANTIC_REFEREE_ENABLED=true`, `SEMANTIC_REFEREE_PROVIDER=anthropic` (or `mcp`) → `{ enabled: false, reason: 'not_implemented' }`, HTTP 200, no key read, no call — the slot is a stub.
7. A room whose GAME-003 `semanticClassification` mode is `'off'` is never sent to the boundary (MCP-004's gate). The boundary's disabled path is the *second* line of defense.

This is the exact behavior `process-language-draft` already ships.

### Client wrapper — `classifyMove`

Added to `src/lib/edgeFunctions.ts`, identical pattern to `processLanguageDraft` (lines 197–226):

```ts
export type ClassifyMoveFunctionResult =
  | { ok: true; data: ClassifyMoveOutcome }
  | { ok: false; error: { error: string }; status: number };

/**
 * Send a redacted move + context to the semantic-referee Edge Function.
 * Returns advisory classification metadata for the caller to reconcile.
 *
 * IMPORTANT: `{ enabled: false }` is a NORMAL, EXPECTED outcome — it is the
 * default state of the whole feature. The caller treats `{ enabled: false }`
 * AND any error identically: fall back to the deterministic layer-1 result,
 * show NO error to the user. Bodies MUST be redacted before calling this.
 */
export async function classifyMove(
  payload: ClassifyMoveRequest,
): Promise<ClassifyMoveFunctionResult> {
  const { data, error } = await supabase.functions.invoke<ClassifyMoveOutcome>(
    'semantic-referee', { body: payload },
  );
  // error → { ok: false, ... } (FunctionsFetchError → status 503);
  // data  → { ok: true, data };
  // empty → { ok: false, error: { error: 'empty_response' }, status: 500 }.
}
```

The error-mapping block is copied verbatim from `processLanguageDraft` (FunctionsHttpError `.context.json()` parse, `FunctionsFetchError` → 503 fallback). The wrapper **never throws**; `{ enabled: false }` and an error both leave the caller on layer 1.

### Node ↔ Deno types reconciliation

Roadmap §6 reconciliation 1: MCP-009 named the Node mirror `src/features/semanticReferee/types.ts`; MCP-011 owns it as `semanticRefereeTypes.ts`. **MCP-016 does NOT create a second Node `types.ts`.** The arrangement:

- **Deno side (documented mirror):** `supabase/functions/_shared/semanticReferee/types.ts` — created by MCP-016. It mirrors MCP-011's `semanticRefereeTypes.ts` (`SemanticRefereePacket`, `SemanticBinarySample`, `Binary01`, the enums, the `ALL_*` constant arrays, `PACKET_VERSION`, `SCORE_HINT_MIN/MAX`) **plus** the boundary types `ClassifyMoveRequest` / `ClassifyMoveRoomContext` / `ClassifyMoveOutcome`. ## Edge cases

The implementer must handle each:

- **Feature flag off** — `{ enabled: false, reason: 'disabled' }`, HTTP 200, no provider selected, no key read. The dominant case.
- **Flag on, provider unset** — defaults to `mock`; deterministic packet, no live call.
- **Flag on, provider `anthropic` or `mcp`** — `{ enabled: false, reason: 'not_implemented' }`, HTTP 200, no key read, no call — the slot is a stub.
- **Flag on, provider an unknown string** — `{ enabled: false, reason: 'not_configured' }`, HTTP 200.
- **No `Authorization` header** — `unauthorized()` (HTTP 401), before any body parse.
- **`Authorization` present but JWT invalid / expired** — `callerClient.auth.getUser()` returns `userError` → `unauthorized()` (HTTP 401).
- **Invalid JSON body** — `badRequest('Invalid JSON body.')` (HTTP 400).
- **Empty `moveBodyRedacted`** — `ClassifyMoveRequestSchema` (`.min(1)`) rejects it → `validation_failed` (HTTP 422). An empty move cannot be classified.
- **Over-length body** — `moveBodyRedacted` / `parentBodyRedacted` > 8000 chars → `validation_failed`. The character cap is the boundary's first guard; the numeric token budget is MCP-004's (`tokenBudget.ts`, merged).
- **`requestedClassifiers` empty or > 5** — schema requires 1–5 → `validation_failed`.
- **`requestedClassifiers` contains an unknown id** — `z.enum(ALL_SEMANTIC_CLASSIFIER_IDS)` rejects it → `validation_failed`. An unmapped id never reaches the mock.
- **No parent** (`parentId` / `parentBodyRedacted` absent) — valid for a root move; the mock derives parent-continuity binaries from layer-1 context and still produces a packet.
- **Caller cannot see the room (RLS)** — the `debates` lookup returns no row → `validation_failed` with `room_not_found_or_not_accessible`. No packet, no provider call.
- **Mock produces a packet that fails its own outbound schema** — should be impossible (the mock is built to be schema-valid), but the function validates the mock's output with `SemanticRefereePacketSchema` before returning; on failure it returns a **deterministic minimal fallback packet** (all binaries `0`, `routeSuggestion: 'no_route_change'`, `frictionSuggestion: 'none'`, all `scoreHints` `0`, `provider: 'mock'`, `authoritative: false`) — never a thrown error. This proves the fallback path even with no live provider.
- **Fixture provider miss** — `contentHash` not in the fixture map → falls through to the mock. Never an error.
- **Concurrent calls for the same move** — the boundary is stateless and idempotent: two identical `ClassifyMoveRequest`s produce two identical mock packets. No lock needed.
- **Offline / network failure between client and Edge Function** — `supabase.functions.invoke` surfaces a `FunctionsFetchError`; the wrapper maps it to `{ ok: false, status: 503 }`; the caller falls back to layer 1. No user-facing error.
- **Doctrine edge case — "what if a future model tries to return a verdict?"** — it cannot, in this card: there is no live provider. And structurally: the outbound `SemanticRefereePacketSchema` is `.strict()` with `authoritative: z.literal(false)` and no truth field — any widened packet fails validation → deterministic fallback. The model physically cannot widen the packet, even after the live pilot lands.
- **Doctrine edge case — "what if disabling the layer breaks the game?"** — it does not: every consumer (MCP-013 ledger, MCP-014 banners) has a layer-1 deterministic fallback. `{ enabled: false }` degrades gracefully.
- **Doctrine edge case — "what if a redacted body still leaks a handle/URL/key?"** — `redaction.ts` runs a **second** defensive pass before the mock sees the body, independent of the client. (The mock does not even forward bodies anywhere — but the pass is structural insurance for when the live provider lands.)

## Test plan

All tests are mock/fixture-driven. **No test makes a live call** (cdiscourse-doctrine §7, test-discipline, roadmap §8.1). Coverage bar: every public function has happy-path **and** failure-case tests. Test files mirror MCP-009 §"Test plan", scoped to the mock-only build.

- **`__tests__/semanticEdgeBoundary.test.ts`** — disabled-by-default. `SEMANTIC_REFEREE_ENABLED` unset / empty / `'false'` → `classifyWithConfiguredProvider` returns `{ enabled: false, reason: 'disabled' }` **and no provider function is invoked** — asserted via a provider spy (the registry calls `runMockClassifier` only after the flag check). The dominant path.
- **`__tests__/semanticProviderRegistry.test.ts`** — registry routing. Flag on, provider unset → `mock` (the doctrine deviation from the language-processing `anthropic` default — its own assertion). Flag on, `SEMANTIC_REFEREE_PROVIDER=mock` → mock. `=fixture` → fixture provider. `=anthropic` → `{ enabled: false, reason: 'not_implemented' }` **and the registry imports no `anthropicProvider`** (the stub is a literal return, no module). `=mcp` → `not_implemented`. Unknown value → `not_configured`.
- **`__tests__/semanticClassifyRequestSchema.test.ts`** — `ClassifyMoveRequestSchema` rejects: empty `moveBodyRedacted`, over-length body (> 8000), empty `requestedClassifiers`, `> 5` classifiers, an unknown classifier id, an unknown extra top-level key (`.strict()`). Accepts a valid request (with and without a parent).
- **`__tests__/semanticPacketSchema.test.ts`** — outbound `SemanticRefereePacketSchema`: accepts a valid packet; rejects `authoritative: true`, a missing `authoritative`, a `value` of `2` / `0.5` / `'1'`, a `scoreHints` integer of `9`, an unknown `classifierId` / `routeSuggestion` / `frictionSuggestion`, a **smuggled `block` field**, a **smuggled truth field**, any unknown extra field. Proves `authoritative` is pinned to literal `false` and `block` / truth fields cannot pass.
- **`__tests__/semanticMockProvider.test.ts`** — `runMockClassifier` is deterministic (same `ClassifyMoveRequest` → byte-identical packet across calls); every packet it produces passes `SemanticRefereePacketSchema`, carries `provider: 'mock'`, `authoritative: false`, `packetVersion: 'mcp-semantic-referee-v0'`; a root-move request (no parent) still yields a valid packet.

## Risks

- **Default-provider divergence.** `languageProcessing/providers.ts` defaults to `'anthropic'`. The semantic referee **must** default to `'mock'`. An implementer copying the language-processing registry verbatim would introduce a doctrine violation. The `?? 'mock'` fallback is load-bearing and has its own test.
- **Accidentally wiring a live provider.** The card's hardest line: the `anthropic` / `mcp` slots must be **literal `{ enabled: false, reason: 'not_implemented' }` returns** with no module behind them. No `anthropicProvider.ts`, no `mcpAdapter.ts`, no `fetch`, no provider host literal. `semanticNoLiveCall.test.ts` enforces this with a source-scan. A reviewer who sees a provider call, a key read, or a deploy step blocks the card (roadmap §7).
- **Two type trees drift.** MCP-001's contract is mirrored in `src/features/semanticReferee/semanticRefereeTypes.ts` (Node, MCP-011-owned) and `supabase/functions/_shared/semanticReferee/types.ts` (Deno, MCP-016-owned). `semanticDenoNodeParity.test.ts` catches drift; a future contract change must update both.
- **Creating a second Node `types.ts`.** Roadmap §6 reconciliation 1 is explicit: MCP-016 imports MCP-011's `semanticRefereeTypes.ts` and does **not** mint a second Node types file. The boundary-only Node types (`ClassifyMoveRequest` etc.) live in `src/lib/edgeFunctions.ts` alongside the wrapper.
- **Zod version.** `process-language-draft` uses `npm:zod@4` inside Deno; MCP-011's validator uses `zod@^4` on the Node side. The new Deno `schema.ts` must use the same `npm:zod@4` specifier — mixing versions in `_shared/` is a footgun.
- **`config.toml` `verify_jwt`.** Forgetting the `[functions.semantic-referee]` block with `verify_jwt = true` would deploy an unauthenticated function. The card must add it.
- **No migration — but verify.** MCP-016 adds no DB change. The implementer must resist adding a packet-cache table; a persisted cache is an MCP-004 decision (MCP-001 §12), a future card's migration.
- **Existing tests.** MCP-016 adds a function, a `_shared` tree, and a wrapper without touching `process-language-draft`, `submit-argument`, or `request-argument-deletion`; no existing test should need updating. The `classifyMove` wrapper is purely additive to `edgeFunctions.ts`.
- **The function exists in code but is not deployed.** Until the operator deploys it, `classifyMove` calls a function that does not exist — `supabase.functions.invoke` returns a 404-class error, the wrapper maps it to `{ ok: false }`, and the caller falls back to layer 1. This is the safe default and is correct: nothing consumes `classifyMove` yet (MCP-013/014 will, later).
