## Goal

Restate the card body verbatim: **the sidecar explains the selected message; the rail owns the main control surface.** SC-003 is the **detail inspector**. SC-004 is the **action dock**. The two cards already exist in the room shell side by side; SC-003 is the refactor that strips the sidecar back to a pure read-only inspector and routes every label through RULE-003, every cluster signal through LIFE-001, every per-move signal through META-001.

The doctrine constraints that shape the design (all from `cdiscourse-doctrine` + the issue body):

1. **No body editing for any user.** Already enforced for `actor === 'self'` bubbles in `ArgumentReplySidecar.tsx` (own-bubble action set is `view_qualifiers` / `request_deletion` only); SC-003 extends this to "no body editing affordance for ANY actor on ANY surface — including the inspector itself."
2. **No internal `snake_case` codes leak to user UI.** Every label is read through `getLifecycleUx(state).label` / `getManualTagUx(code).label` / `getAutoMetadataUx(code).label`; the inspector never re-authors a label.
3. **Timeline mode is concise; Stack mode reveals deeper semantic flags.** A `viewMode: 'timeline' | 'stack'` switch on the view-model builder gates the per-flag chip strip.
4. **Sidecar never carries the action palette.** SC-003 builds a view model with **no action callbacks** and **no action codes**. SC-004 owns that surface; SC-003 enforces the boundary at the type level (`argumentReplySidecarModel.ts` does not value-import or type-import from `timelineNodeActionDockModel.ts`).
5. **Score is never truth.** Standing / tone / heat chips already on the sidecar are presented as game-state, not verdicts; the view model surfaces them through the existing `standingBandCopy.formatStandingBandShort` helper, unchanged.

## What SC-003 IS and IS NOT — boundary against SC-004

### SC-003 is

- A typed view-model + a pure-render component that surfaces **6 sections** for the currently-selected move:
  1. **What this move says** — the body excerpt + actor + side + timestamp + parent hint.
  2. **Why it matters** — the cluster's RULE-003 lifecycle helper line.
  3. **What is unresolved** — open auto-metadata items + open source-chain / quote requests on this move's cluster.
  4. **Where it sits** — branch / lane / depth / path through the tree.
  5. **Suggested next move** — a typed STUB slot (`SuggestedMoveStub | null`) reserved for ST-002 (issue #13). SC-003 always returns `null` here.
  6. **Semantic flags (deeper in Stack mode)** — manual-tag + auto-metadata chips. In Timeline mode: only a count + "Show details" affordance. In Stack mode: the full chip strip.

### SC-003 is NOT

- The action dock. **All action affordances must be removed from `ArgumentReplySidecar.tsx`** — the existing `OTHER_ACTIONS` / `SELF_ACTIONS` arrays (lines 32–48) and the `actionsRow` (lines 155–168) move into SC-004's `TimelineNodeActionDock` component, which the room shell (`ArgumentGameSurface.tsx`) already renders directly above the sidecar (see lines 504–518 of the current room shell).
- A composer / submit path. The view model exposes no callback that a user can press to post.
- A lifecycle deriver. SC-003 reads `PointLifecycleClusterSummary.state` from the supplied lifecycle map; it never calls `buildPointLifecycleMap`.
- A label authoring surface. RULE-003's `LIFECYCLE_UX_MAP` / `MANUAL_TAG_UX_MAP` / `AUTO_METADATA_UX_MAP` are the only source of truth for chip labels and helper lines.

### SC-003 ↔ SC-004 boundary (test-enforced)

| Concern | Owner | Forbidden in other |
|---|---|---|
| Action vocabulary (`TimelineNodeActionDockActionCode`) | SC-004 | SC-003 must not value-import or type-import from `timelineNodeActionDockModel.ts`. |
| Cluster lifecycle label / helper / iconHint | RULE-003 | Neither SC-003 nor SC-004 re-authors these strings. |
| Per-move auto-metadata chip / manual-tag chip | RULE-003 | SC-003 reads through `getAutoMetadataUx` / `getManualTagUx`. |
| Body excerpt rendering | SC-003 | SC-004 never renders the body. |
| Composer preset round-trip (`actionDockToComposerPreset`) | SC-004 | SC-003 never produces a composer preset. |
| Suggested next move derivation | ST-002 (future) | Both SC-003 and SC-004 reserve a typed slot only. |

The boundary is enforced two ways:
- A source-scan test asserts `argumentReplySidecarModel.ts` and `ArgumentReplySidecar.tsx` contain zero references to `timelineNodeActionDockModel`, `TimelineNodeActionDockActionCode`, `actionDockToComposerPreset`, or the string literal `'expand_branch'`.
- A type-level test (a `.test-d.ts` style assertion using `@ts-expect-error`, or a runtime `Object.keys(vm).every(k => !k.toLowerCase().includes('action'))` scan) asserts the view-model shape has no field whose name contains `action`, `dispatch`, `onPress`, `onAction`, or a callback type.

## API / interface contracts

### Builder contract

```ts
buildSidecarViewModel(input: BuildSidecarViewModelInput): SidecarViewModel
```

- **Pure.** No side effects. Same input → same output. No `Date.now()`, no `Math.random()`, no mutation of input arrays.
- **Total.** Every input produces a structurally valid `SidecarViewModel`. Empty / null inputs produce the `isEmpty: true` shape; never throws.
- **Single dependency boundary.** The builder may value-import from:
  - `./argumentGameSurfaceModel` (surface model types).
  - `../lifecycle` (LIFE-001 barrel — types + `getPointLifecyclePlainLabel`).
  - `../metadata` (META-001 barrel — types + `getManualTagPlainLabel` / `getAutoMetadataPlainLabel`).
  - `../rulesUx/lifecycleUxMap` (RULE-003 — `getLifecycleUx`, `getManualTagUx`, `getAutoMetadataUx`).
  - `./standingBandCopy` (existing helper; for band labels on the body section).
- **Forbidden imports** (test-enforced):
  - `./timelineNodeActionDockModel` (any path).
  - Any path under `supabase`, `app/`, `expo`, `react`.
  - `./quickActionPresets` (composer seam).
  - `./conversationMoves` (composer seam).
  - `./composerHandoff` / `./composerState` / `./composerSubmit` / `./composerValidation`.
- **No callback fields.** The returned shape has zero fields of type `function` and zero fields whose name is `onPress`, `onAction`, `dispatch`, `submit`, etc. The component derives all interactivity from props external to the view model (and SC-003's component has none — it is purely presentational).

### Component contract

```ts
interface ArgumentReplySidecarProps {
  viewModel: SidecarViewModel;
}
export function ArgumentReplySidecar(props: ArgumentReplySidecarProps): JSX.Element;
```

- One prop. No optional callbacks.
- Pure render. No internal state. No effects beyond reading `viewModel`.
- Accessibility: the root view exposes `accessibilityLabel={viewModel.accessibilityRootLabel}`; each section is wrapped in a `<View accessible accessibilityRole="summary">` (or `"text"` on platforms where `summary` is not supported, gated by `Platform.select`).
- `testID="argument-reply-sidecar"` stays.

20. Body length 500 chars → `bodyExcerpt.length <= 281` (280 + ellipsis), `isTruncated === true`, last char is `…`, word-boundary respected.
21. Body 500 chars with no whitespace → hard-truncate at 280, `isTruncated === true`.
22. Soft-deleted move → `isHidden === true`, `bodyExcerpt === ''`.

**Mode toggle**
23. Build VM with `viewMode: 'stack'`, then again with `viewMode: 'timeline'`; assert section count and ordering are identical; assert `SemanticFlags.isCondensed` flips.

**Determinism**
24. Build twice with identical inputs → deep-equal output (no `Date.now()`, no randomness).

**No-action contract**
25. Iterate every key in the returned VM (recursive) → no key name matches `/action|dispatch|onPress|onAction|submit/i`. Source-scan assertion.
26. Iterate every value type in the VM → no value is a `function`. Runtime + type assertion.

**Snake_case ban**
27. `JSON.stringify(viewModel)` against every state combination → assert no rendered field value matches `/[a-z]+_[a-z]+/` EXCEPT for explicit `sourceCode` fields (which are marked "NEVER rendered" and exist for tests + ST-002). A whitelist of field paths is maintained in the test.
