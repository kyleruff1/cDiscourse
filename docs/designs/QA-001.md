## Goal (one paragraph)

QA-001 turns the Design Pass's ten "Core user journeys" (¬ß4, J1‚ÄìJ10) into the ASP program's release gate: one **operator-armed manual checklist** that scripts each journey step-by-step with expected states, plus the **automated jest coverage** for every journey seam that is testable today without live auth. The card ships regression protection *with* the surface overhaul instead of as a follow-up (test-discipline: tests are part of "done"). It is doctrine-bound the same way the surfaces are ‚Äî the gate itself asserts the non-negotiables along every journey: no verdict/truth/score copy in any visible string, exactly one state per node (no chip-soup), a private room never exposes a chime-in/floor affordance, and the mediator board is derived once and shared. Three journeys (J5 record-voice, the audio half of J6, J8 save-recording) have **no shipped surface** ‚Äî they are blocked on VOICE-ADR-002 (#863) ‚Äî so the gate documents them as unarmed and the automated manifest asserts they claim no test and carry the #863 marker. This is a **tests + docs only** card: zero `supabase/` changes, no new deps, no engine edits, and every pinned production file stays byte-identical.

## Cannot-proceed check

No blocking conflict. The card is well-specified after the 2026-07-08 re-scope. Three interpretation points require an orchestrator ruling (they do not block design ‚Äî defaults are chosen and justified below, see **¬ß Reconciliation points**): (1) checklist doc placement, (2) automated-spine scope (gap-fill vs full 8-file symmetry), (3) whether the release gate may be "conditionally green" with the three voice journeys blocked. The Design Pass ¬ß4 journey prose also uses *aspirational* copy that differs from shipped strings (e.g. "Respond to this moment" vs shipped `'Respond to this'`; "1 receipt owed"/"Receipts?" vs shipped `'Source owed'`); the gate scripts are authored against **shipped** copy, and the divergences are logged in the ledger.

## API / interface contracts

Nothing another production file calls ‚Äî QA-001 exposes no runtime surface. The contracts below are the **test-internal** shapes the implementer builds against.

### 1. Journey manifest (inside `journeyGateCoverageMap.test.ts`)

```ts
type JourneyId = 'J1'|'J2'|'J3'|'J4'|'J5'|'J6'|'J7'|'J8'|'J9'|'J10';

interface JourneyManifestRow {
  id: JourneyId;
  title: string;                 // short journey name
  flag:                          // the ASP flag the journey needs
    | 'home_v2' | 'room_exchange_v2' | 'proof_drawer'
    | 'voice_entries' | 'timestamp_rebuttals' | 'one_time_playback'
    | 'move_marks' | 'none';
  flagLiveState: 'live' | 'off'; // production runtime state, NOT the code default (code default is OFF for all 8)
  blockedOn: string | null;      // '#863' for the voice journeys, else null
  // Relative paths under repo root. Empty for blocked journeys.
  spineOfRecord: readonly string[];
  scriptSection: string;         // anchor into docs/qa/journey-gate-j1-j10.md, e.g. '#j1'
}

const JOURNEY_MANIFEST: readonly JourneyManifestRow[]; // exactly 10 rows, frozen
```

Guard assertions: exactly 10 rows; ids are J1..J10 with no gaps/dupes; every non-empty `spineOfRecord` path resolves on disk (`fs.existsSync`); every `blockedOn === '#863'` row has `spineOfRecord.length === 0` and `flag ‚àà {voice_entries, one_time_playback, timestamp_rebuttals}`; `flagLiveState` for `home_v2` and `room_exchange_v2` is `'live'`, all others `'off'`. Firing negative control: a synthetic row pointing at `__tests__/__does_not_exist__.test.ts` must fail the existence assertion (proves the guard can fail).

### 2. Consolidated ban-list (inside `journeyGateDoctrineBanList.test.ts`)

Reuses the house convention verbatim (`copySystemBanList.test.ts` pattern): a `bannedTokens()` array (the verdict/AI-authority/social-feed family plus the surface-specific bans `'proof'`, `'proven'`, `'validated'` from `proofDrawerCopyBanList`), a recursive `flattenStrings(value)`, and a word-boundary `matchesBannedToken(text, token)`. The scanned corpus is the union of the shipped copy constants each journey renders:

Positive control array (planted): `['This is the winner.', 'This is proof of it.', 'Join the debate']` ‚Äî each must trip. ### 3. Spine-test contract (both new spines)

A spine test imports the **real** pure models, composes them in user-visible order, and asserts (a) the ordered handoff ‚Äî step N's output is the input the UI feeds step N+1 ‚Äî and (b) the visible verb/copy/testID string at each step. A spine test **does not** re-assert unit branch coverage; its file header names the unit-suite owner. No React Native render is required for J9 (pure-model composition); J6-text may use one RNTL assertion for the phrase-picker ‚Üí chip render but leans on the model.

- **Empty inputs.** J1 first-run (both strips empty) is the primary empty-state; the spine/coverage tests must not assume any seeded room. `segmentPhrases('')` ‚Üí `[]` (J6); `buildArgumentHomeViewModel` with no debates ‚Üí `isFirstRun: true`.
- **Blocked journeys have no surface.** J5/J8/J6-audio: the manifest must assert `spineOfRecord.length === 0` AND the `#863` marker ‚Äî do **not** stub a fake voice surface to make a green cell. A blocked cell is an honest "not yet," not a skipped test.
- **Copy divergence (shipped vs Design Pass prose).** Author scripts against **shipped** strings (`'Respond to this'`, `'Source owed'`, `'Answer this ‚Üó'`), not the aspirational ¬ß4 prose. The consolidated ban-list scans shipped constants; if a future card renames a constant, the ban-list import breaks loudly (good).
- **`deriveConversationEntryHint` is renamed.** The live export is `deriveGalleryEntryHint` (GAL-002). Any test/doc referencing the old name must use the new one; the old name survives only in comments.
- **Span drift (J6).** `buildSourceSpanSegments` returns `null` when offsets no longer index the body ‚Üí the chip must fall back to `quoted_text` with no highlight. The spine must assert the `null` branch *and* that `formatMarkerChipLabel(quoted_text)` still yields a chip label.
- **Debt not discharged by a bare quote (J7).** A quote-only attach must keep the debt `requested` (only `moveSuppliesKind` kinds discharge). The J7 spine-of-record already asserts this negative ‚Äî the coverage map must keep citing it.
- **One state per node (J9).** An ordinary open node has **zero** chips; `getNodeMediatorMarker` returns `null`. The spine must assert a plain node yields no marker (no chip-soup), not just that a marked node yields one.
- **Never-required feedback (J10).** The J10 spine-of-record asserts the mark surface does not touch submit/compose. The coverage map must keep this citation ‚Äî it is the doctrine anchor "score never blocks posting."
- **Permission-denied / observer paths.** J10 bar renders inert for own-move and for observers; J9 parity holds across viewer roles. These are covered by existing actor-matrix suites (`moveMarksCapabilityParity.test.ts`, `roomCapabilityParityMatrix.test.ts`) ‚Äî cite, don't duplicate.
- **Offline / no-`.env`.** The manual checklist must carry the "no `.env` path" fallback (mirrors `mvp-smoke-test.md`): with no Supabase config, only the flag-off byte-identity and render-shape steps are testable; auth-bearing steps are recorded as skipped. Automated tests never need `.env` (pure models + JSDOM).
- **Manifest file-rename coupling.** If a cited spine file is renamed, `journeyGateCoverageMap.test.ts` fails the `existsSync` assertion by design ‚Äî the fix is to update the manifest + the doc table in the same commit (this is the doc-drift protection, not a bug).

## Risks

- **Spine tests turning tautological.** The sharpest risk. A spine that just re-asserts a fixture echoing a fixture adds nothing. Mitigation: the two NEW spines assert the **ordered handoff** (step N's output is literally the input to step N+1) and the **cross-model identity** (J9: the same frozen board object flows to rail + node markup; the `answerThis` target equals the tapped node id), which no unit suite asserts. Where an existing flow test already walks the full ordered sequence (J1/J2/J3/J4/J7/J10) the design **forbids** a duplicate spine and cites the existing file instead. If the reviewer judges a NEW spine still overlaps its unit owners, collapse it to a citation.
- **Coverage-map guard brittleness.** `existsSync` on cited filenames fails on legitimate renames. This is intended (doc-drift protection) but adds a maintenance step. Mitigation: the manifest and the doc table live in the same doc-section/commit; the maintenance rule is stated in both the doc and the test header.
- **Flag-status drift.** `home_v2`/`room_exchange_v2` are LIVE, the rest OFF ‚Äî this will change as flips land. The gate's flag-status column and the manifest's `flagLiveState` will go stale. **Maintenance rule (stated in the doc):** update the flag-status column *and* `flagLiveState` at each flip, in the flip's PR. Consider whether the manifest should assert live-state at all (it couples tests to ops state) ‚Äî see reconciliation point 3.
- **Existing perf flakes (`pointLifecycleModel`, `moveMetadataLedger`).** A full-suite run may show a red in one of these unrelated to QA-001. Re-run isolated before blaming the branch (documented in the test plan).
- **Copy-constant renames break the ban-list import.** If a future card renames `HOME_COPY` etc., `journeyGateDoctrineBanList.test.ts` fails to compile. This is acceptable (loud, early) but means the ban-list must be updated with any copy refactor.
- **Manual-checklist rot.** The operator script can drift from the shipped UI. Mitigation: keep the script anchored to shipped **testIDs** (which the automated suites also assert), so a testID rename surfaces in both places.
