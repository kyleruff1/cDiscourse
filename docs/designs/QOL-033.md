**Depends on:** QOL-030 (the popout chassis).

## 0. How this design was built

Derived — **node stages in timeline mode** define what Go can filter and jump
to (§3); the **narratives** prove Go is needed exactly when an argument grows
past one screen (§5). Go is the *traverse / re-view* surface — it moves the
viewport and reconfigures the board; it never authors and never inspects detail.

## 1. Goal & scope

The **Go popout** is the navigate-and-re-view surface. It converges:

- **IX-002** timeline mini-map (shipped)
- **IX-001** density modes + focus lenses (in flight — designed in here)
- the **Timeline ↔ Cards** view toggle

In scope: jump targets, density, focus lenses, the embedded mini-map, the view
toggle, and how all of these read **node stages**. Out of scope: the chassis
(QOL-030); the mini-map's internal projection model (`timelineMiniMapModel` —
consumed); per-node detail (that is Inspect, QOL-032).

## 2. Inputs

QOL-030's chassis. Read-only: `timelineMiniMapModel` (IX-002), the IX-001
density hooks already used by VG-004, LIFE-001 node stages, BR-001/004 branch
topology. Go **reconfigures the viewport and the board's render mode** — it
performs no write and no content mutation.

## 3. The Go popout in timeline mode

### 3.1 Anchoring

Unlike Act and Inspect, Go is **not** anchored to a node — it is about the whole
board. It anchors to a fixed board corner. It opens fast, never hides the board,
dismisses on `Esc` / scrim / selection.

### 3.2 The four controls

| Control | Entries | Effect |
|---|---|---|
| **Jump** | Root · Latest · Hot zone · Branch list | pans / focuses the board — **no route transition** |
| **View** | Timeline ⇄ Cards | the existing view toggle (re-presents; keeps active node) |
| **Density** | Compact · Normal · Expanded | IX-001 density; persists per session; preserves the active node |
| **Lens** | Active path · Unresolved · Evidence | a focus filter — dims non-matching nodes; **hides nothing, data intact** |

### 3.3 Lenses are stage filters

The focus lenses are defined **by node stage** — this is the node-stage basis of
Go:

| Lens | Matches nodes at stage |
|---|---|
| **Active path** | the path from root to the active node (topology, not stage) |
| **Unresolved** | `open` · `rebutted` · `clarified` · `source_requested` · `quote_requested` · `narrowed` |
| **Evidence** | `source_requested` · `quote_requested` · `sourced` |

A lens **dims** non-matching nodes to a faint state — it never removes them and
never blocks navigation to them. "Hot zone" (Jump) is the longest run of warm/
hot nodes by **activity**, never by correctness (IX-002 doctrine, preserved).

### 3.4 The embedded mini-map

IX-002's mini-map renders inside the Go popout as a one-strip overview: one
marker per node, the viewport window, the branch clusters, the hot zone. Tapping
a mini-map region pans the board — no route transition. The mini-map is a
**projection** — it re-derives nothing (IX-002's locked doctrine).

## 4. Narrative frames — Go in action

| Frame (scenario) | Why Go is used | Go does |
|---|---|---|
| S1 F5 — Observer C arrives at a busy public room | C must survey before chiming in on the right node | **Jump → Hot zone**; **Lens → Unresolved** to see where a contribution lands cleanly |
| S1 F10 — the argument has a mainline + a tangent branch | A needs to keep the dishes line and the door-tangent both legible | **Branch list**; **Lens → Active path** to stay on the mainline |
| S1 F20 — settlement, long board | confirm nothing is left open before settling | **Lens → Unresolved** shows zero dimmed-in nodes → safe to settle |
| S2 F5 — "Evidence requested" | A wants to see every owed source at a glance | **Lens → Evidence** highlights the `source_requested` node |
| 250-message stress room | the board is a very wide scroll | **mini-map** + **Density → Compact** keep the whole argument legible |

Go matters precisely when the argument outgrows one screen — every frame above
is a real storyboard moment, not a hypothetical.

Refines [`one-box-interface-model.md`](../ux-storyboards/one-box-interface-model.md) W8:

```
 ╔════════════════════════════════╗
 ║ GO                       · esc ║
 ║ JUMP    R root    L latest     ║
 ║         Z hot     B branches   ║
 ║ VIEW    T  timeline ⇄ cards    ║
 ║ DENSITY 1 compact 2 normal     ║
 ║         3 expanded             ║
 ║ LENS    P active-path          ║
 ║         U unresolved ◀ on      ║
 ║         E evidence             ║
 ║ ▁▂▃▅▇▅▂▁▂▃▁  ← mini-map        ║
 ║  └ viewport window ┘           ║
 ╚════════════════════════════════╝
```

## 6. Edge cases

- **Short argument (< 12 nodes)** — the mini-map strip is omitted (IX-002's
  threshold); Jump / View / Density / Lens still render.
- **No branches** — the Branch-list entry is disabled with the reason "no side
  branches yet".
- **A lens that matches zero nodes** — the popout shows "nothing unresolved" /
  "no evidence yet"; the board is left unfiltered (dimming nothing).
- **Density change** — the active node is preserved on screen (IX-001 contract).
- **Cards view** — Density and the mini-map are timeline-only; in Cards view
  those entries are disabled with the reason "available in Timeline view".

## 7. Test plan

- Jump targets resolve to the right node; no route transition occurs.
- Lens membership — the §3.3 stage tables; a lens dims, never removes; data is
  intact behind a lens.
- Hot zone is chosen by activity length, never by standing/heat-as-correctness.
- Density preserves the active node; persists per session.
- Mini-map omitted below the node threshold; embedded above it.
- Cards-view disabled-with-reason for timeline-only controls.

## 8. Doctrine & safety self-check

- A lens **dims, never hides** — all nodes stay navigable, all data intact.
- Heat / hot zone is an **activity** signal — never correctness; the word
  "hot" never co-occurs with a verdict.
- Go performs no write, no content mutation — viewport + render-mode only.
- Deterministic pure projection over existing models — no AI, no network.
- No verdict / winner / loser copy in any label.

## 9. Dependencies & do not implement

- **Depends on:** QOL-030 (chassis); consumes IX-002 `timelineMiniMapModel`,
  IX-001 density hooks, LIFE-001 stages, BR-001/004 topology — all read-only.
- **Do not implement here:** the chassis; IX-002's projection model; per-node
  detail (QOL-032 Inspect owns that).
