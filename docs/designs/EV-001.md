## Goal

Promote evidence from a transient form field on the composer (today: `attached_evidence: { url?, label?, source_text? }[]`, kept only inside the `client_validation` / `server_validation` JSONB snapshots) into a first-class, addressable **gameplay object** with a stable identity, a typed `kind`, a deterministic `sourceChainStatus`, an `EvidenceRisk` band, and a clear receipt-chip + timeline-node display contract.

EV-001 is the foundation block of Epic 6:

- **EV-002 (source-chain popover)** reads `sourceChainStatus` to pick which non-accusatory prompt to render.
- **EV-003 (evidence debt tracker)** reads the artifact list to decide which `OpenIssueDebt` rows are still open vs resolved by later moves.
- **EV-004 (rule symmetry)** consumes the artifact `kind` + status to map between the validator's `evidence_required` / `evidence_debt` codes and the user-facing tools in `ruleToUiMap.ts`.

EV-001 itself ships **no popover, no debt counter, no rule-symmetry UI**. It ships:

1. A pure-TS data model (`EvidenceArtifact`, four enums, two pure helper functions).
2. A read-time adapter that normalizes the existing `attachedEvidence[]` payload into `EvidenceArtifact[]` — **zero schema change in v1**.
3. A receipt-chip display contract (props + label rules, no React in this card's model layer).
4. A timeline-node evidence/source-chain status contract that visual-grammar renderers can layer on.
5. Reinforcement of the doctrine that **missing evidence hard-blocks only `argument_type = 'evidence'` posts**; ordinary replies stay postable. The existing `EVIDENCE_SOURCE_REQUIRED` rule in `src/domain/constitution/evaluateArgumentDraft.ts` already encodes this; EV-001 documents the contract and asserts it via tests.

Doctrine anchors that shape the design:

- **cdiscourse-doctrine** §1 — Score / status is gameplay analysis, never truth. Status fields describe the *trail*, not the *truth*.
- **cdiscourse-doctrine** §3 — Popularity is not evidence. `EvidenceArtifact.kind` does not include "many users agree", "retweet count", "view count", or "verified author".
- **evidence-doctrine** "What counts" — exactly six `kind` values, all inspectable artifacts.
- **evidence-doctrine** "Source-chain status" — five-state ladder; broken-chain and primary-source are real states the UI must render.
- **supabase-edge-contract** §1 — No service-role, no direct insert. EV-001 v1 changes neither — adapter-only.
- **test-discipline** — Pure-model tests for every public function + ban-list test for receipt-chip strings.

## Persistence decision (EXPLICIT)

**Chosen path: (b) — Pure TypeScript adapter, no schema change in v1.**

EV-001 v1 normalizes the existing `attached_evidence: [{ url?, label?, source_text? }]` payload (read from `arguments.client_validation.attachedEvidence` / `arguments.server_validation.attachedEvidence` / live `ComposerDraft.attachedEvidence`) into `EvidenceArtifact[]` at read time, in a pure-TS adapter (`buildEvidenceArtifacts(input)`).

### Why (b), not (a) or (c)

- **Not (a) — pure UI/model only:** The existing payload is structurally too thin. It has no stable `id` (we have to mint one deterministically), no `kind`, no `sourceChainStatus`, no `risk`, and no `addedByUserId` / `createdAt` (those live on the parent `arguments` row). EV-002/3/4 need a stable record shape they can consume; ad-hoc reads from JSONB are a tax we'd pay forever.
- **Chose (b) — adapter:** The adapter derives `id` deterministically (`<argumentId>:evidence:<index>`), classifies `kind` from the populated fields (URL present → `url`; `source_text` only → `source_text`; both → still surfaces as multi-kind through the artifact list, see §"Adapter rules"), runs the `sourceChainStatus` decision table over the artifact (default: `unverified` for any user-attached artifact; never auto-promotes to `primary_source`), and assigns `risk = 'unknown'` by default. Every field is reproducible from existing rows.
- **Not (c) — new migration now:** EV-001 is one card in a four-card epic. Committing a migration before EV-002/3/4 settle (a) blocks the operator on a deploy step, (b) risks needing a second migration to add columns EV-003 turns out to want (per-debt resolution, anti-amplification flags), (c) violates the "small, reversible, testable" cadence the agent workflow assumes. The design records a **path (c) follow-up card** (see §"Build sequencing") that the operator can schedule when the artifact shape stabilizes after EV-003.

### What this means in practice

- **No new migration files in EV-001.**
- **No new Edge Function in EV-001.**
- **No new RLS policy in EV-001.**
- **No new Supabase write.**
- `submit-argument` continues to receive `attached_evidence` as it does today; the engine still consumes `EvidenceAttachment` exactly as documented in `src/domain/constitution/types.ts`. EV-001 adds an adapter *layer* on top — nothing under it changes.
- Operator runs **no deploy step** for EV-001.

### What the follow-up (path c) card would do later

After EV-003 stabilizes the artifact shape, a follow-up card may add:
- `public.argument_evidence_artifacts` (id, argument_id FK, kind, label, url, source_text, quote, source_chain_status, risk, added_by_user_id, created_at) with RLS mirroring `arguments` visibility.
- A migration that backfills from existing `arguments.*_validation.attachedEvidence` JSONB snapshots using the adapter logic (so the adapter stays the authoritative classifier).
- An update to `submit-argument` to insert artifact rows alongside the argument.

The adapter built in EV-001 is the contract that follow-up consumes — read-time code does not change, only the source it reads from.

## Data model

All public types live in a new pure-TS module `src/features/evidence/evidenceModel.ts`. No React. No Supabase. No network.

```ts
/**
 * The kind of inspectable artifact attached to a move. New kinds are added
 * here; no UI surface may invent its own. Popularity, author identity,
 * follower count, and "verified" status are deliberately NOT kinds — see
 * evidence-doctrine "What does NOT count as evidence."
 */
export type EvidenceArtifactKind =
  | 'url'                  // Inspectable web URL the user pasted.
  | 'quote'                // Verbatim text excerpt; should match a linked source.
  | 'source_text'          // Free-text excerpt of a source (no URL).
  | 'dataset'              // Pointer to a dataset (URL or identifier).
  | 'screenshot_redacted'  // Image artifact with PII / handles stripped.
  | 'manual_citation';     // Bibliographic-style citation (book, paper, broadcast).

/**
 * Status of the chain from claim → cited evidence. Strictly advisory; the
 * UI uses it to pick the non-accusatory prompt, never to declare truth.
 *
 *   unverified         user attached but nothing inspected yet
 *   source_no_quote    URL or citation present, but no quote / excerpt
 *   source_and_quote   URL/citation AND a quote / excerpt are both present
 *   broken             chain is dead (404, contradictory, circular) — set
 *                      manually or by a later admin check; EV-001 never
 *                      auto-decides this
 *   primary_present    chain reaches an inspectable primary record — set
 *                      manually by an admin or by a future automated check;
 *                      EV-001 NEVER auto-promotes to this value
 */
export type SourceChainStatus =
  | 'unverified'
  | 'source_no_quote'
  | 'source_and_quote'
  | 'broken'
  | 'primary_present';

/**
 * How confidently the system treats this artifact's contribution to the
 * argument's standing. NOT a truth claim about the source — describes the
 * INSPECTABILITY of the artifact. Always advisory.
 */
export type EvidenceRisk = 'low' | 'medium' | 'high' | 'unknown';

/**
 * The full artifact record. v1 produces this from the adapter; later
 * persistence (path c) would store rows of this shape.
 */
export interface EvidenceArtifact {
  /** Stable deterministic id. Adapter mints: `<argumentId>:evidence:<index>`. */
  id: string;
  /** FK back to public.arguments.id. */
  argumentId: string;
  kind: EvidenceArtifactKind;
  /** Plain-language label the user typed or the adapter derived. Max 120 chars. */
  label: string;
  /** Optional URL — present when kind = 'url' or 'dataset'. */
  url?: string;
  /** Optional source text excerpt — present when kind = 'source_text'. */
  sourceText?: string;
  /** Optional verbatim quote tied to the source. */
  quote?: string;
  sourceChainStatus: SourceChainStatus;
  risk: EvidenceRisk;
  /** The author of the parent argument. Adapter copies from arguments.author_id. */
  addedByUserId: string;
  /** ISO-8601. Adapter copies from arguments.created_at. */
  createdAt: string;
}
```

Note on plurality: a single move may have **zero, one, or many** artifacts. The adapter returns an array (possibly empty). Receipt-chip rendering (see §"Receipt-chip display contract") summarises the whole array into one chip.

### `SourceChainStatus` derivation (decision table)

Pure function, no I/O:

| URL present | sourceText present | quote present | derived status         |
| :---------: | :----------------: | :-----------: | ---------------------- |
|     no      |        no          |      no       | (no artifact emitted)  |
|     yes     |        no          |      no       | `source_no_quote`      |
|     no      |        yes         |      no       | `source_no_quote`      |
|     yes     |        yes         |      no       | `source_no_quote`      |
|     yes     |        no          |      yes      | `source_and_quote`     |
|     no      |        yes         |      yes      | `source_and_quote`     |
|     yes     |        yes         |      yes      | `source_and_quote`     |
|     no      |        no          |      yes      | `unverified` (quote without source pointer is structurally weak) |

`broken` and `primary_present` are NEVER auto-derived in EV-001. They are reserved for admin / future automated checks. Adapter callers may pass an `overrides: { [artifactId: string]: Partial<EvidenceArtifact> }` to override `sourceChainStatus` / `risk` (used by tests + future admin tooling); v1 production never uses this.

`EvidenceRisk` default in v1: `'unknown'`. Reserved for an EV-003-era risk classifier; v1 leaves the field present but the classifier is out of scope.

export interface EvidenceAttachmentInput {

  label?: string | null;
  sourceText?: string | null;
  quote?: string | null;
  /** Optional explicit kind. When omitted, classifyEvidenceKind derives it. */
  kind?: EvidenceArtifactKind;

  argumentId: string;
  addedByUserId: string;

 *  quote without any source pointer is downgraded to `source_text`. */

export function deriveSourceChainStatus(
  att: Pick<EvidenceAttachmentInput, 'url' | 'sourceText' | 'quote'>,

  helper: string;
  /** Logical tone — renderers map this to a color token. NOT a truth label. */

  count: number;

### Adapter rules (`buildEvidenceArtifacts`)

1. **One artifact per non-empty attachment.** An attachment is "empty" if `url`, `sourceText`, and `quote` are all null / blank / undefined; it is dropped.
2. **Deterministic id:** `${argumentId}:evidence:${index}` (index is the 0-based position in the original `attachments` array, ignoring dropped entries). Stable across reads.
3. **Label fallback chain:** explicit `label` → hostname of `url` → first 32 chars of `sourceText` → "Attached evidence". Truncate to 120 chars.
4. **Kind classifier (`classifyEvidenceKind`):**
   - explicit `att.kind` wins (used by future ingestion paths);
   - else `url` present:
     - `url` matches dataset host allowlist (e.g. `data.gov`, `*.figshare.com`, `*.zenodo.org`) → `dataset`;
     - else → `url`;
   - else `sourceText` present → `source_text`;
   - else `quote` present alone → `source_text` (treated as the text excerpt of an un-located source);
   - else (shouldn't reach here because the entry would have been dropped) → fallback `manual_citation`.

   *Note:* `screenshot_redacted` and `manual_citation` are reserved for future ingestion paths (uploads, bibliographic forms) — v1 adapter never emits them from the current attachment shape. Tests assert the enum contains both so EV-002+ can already build for them.

5. **Status:** `deriveSourceChainStatus({ url, sourceText, quote })` per the decision table above. `broken` and `primary_present` only set via `overrides`.
6. **Risk:** default `'unknown'`. Set via `overrides` only.
7. **`addedByUserId`, `createdAt`:** copied from the `BuildEvidenceArtifactsInput`. Adapter does NOT read Supabase. Caller is responsible for sourcing them from `arguments.author_id` / `arguments.created_at`.
8. **Overrides merge:** `overrides[artifactId]` is shallow-merged after derivation. Used by tests; production calls omit.

### How EV-002, EV-003, EV-004 will consume EV-001

Naming the exact exports each downstream card will depend on (so EV-001's surface stays stable):

- **EV-002 (source-chain popover)** will:
  - read `EvidenceArtifact[]` for the active node;
  - call `summarizeArtifactsForReceiptChip(artifacts)` and dispatch on `contract.status` to pick its popover copy ("Ask for source" / "Ask for quote" / "Inspect receipt" / "Source trail is weak" / "Source trail anchored");
  - read `contract.invitesFollowup` to decide whether to render the composer-trigger CTA;
  - never re-derive the status itself.

- **EV-003 (evidence debt tracker)** will:
  - read `EvidenceArtifact[]` per move to decide which `OpenIssueDebt` rows (from `src/features/pointStanding/types.ts`) of types `source_needed`, `quote_needed`, `primary_record_needed` are resolved by a later move attaching the matching artifact;
  - use `kind` + `sourceChainStatus` as the close-condition (`source_needed` → any artifact with status ≥ `source_no_quote`; `quote_needed` → status ∈ {`source_and_quote`, `primary_present`}; `primary_record_needed` → status === `primary_present`);
  - never mutate the artifact; debts close by emitting new `PointStandingDelta`s.

- **EV-004 (rule symmetry)** will:
  - extend `src/features/rulesUx/ruleToUiMap.ts` so the `evidence_debt` and `source_chain` affordances cross-link to the receipt-chip contract (e.g. clicking "Needs receipts" in the sidecar opens the composer pre-set to attach an artifact);
  - assert that the chip's plain-language strings + the rule-map's affordance strings stay consistent through a shared ban-list test.

## Edge cases

The implementer must handle each:

1. **Empty attachments array.** Adapter returns `[]`. Chip renders the "No receipts yet" form. Timeline contract renders no ring on a non-evidence node. An *evidence* node with empty artifacts is the existing validator block (`EVIDENCE_SOURCE_REQUIRED`); the chip still renders "No receipts yet" defensively in case the row sneaks in.
2. **Attachment with all fields null / blank.** Dropped. Index for the next non-empty attachment is its original position, NOT renumbered (stability promise: same input → same ids on re-read).
3. **Multiple URLs in one attachment.** Adapter sees one `url` field; it does NOT split. Multiple URLs are multiple attachments.
4. **Quote without source pointer.** Classified as `source_text` (kind), status `unverified` (table row 8). Chip says "Receipt attached" / "Pointing to a source would strengthen it."
5. **URL pointing to a known dataset host.** Classified `dataset`. Status still uses the (url, sourceText, quote) decision table — `dataset` is a kind, not a status.
6. **Concurrent edits.** Not applicable in v1 — the adapter is read-only and pure. EV-003 may need a debt-resolution race story, but that's its problem.
7. **Offline / network failure.** Not applicable — adapter does not read Supabase.
8. **Permission-denied paths.** RLS on `public.arguments` already gates whether the row (and therefore the embedded validation snapshot) is visible. The adapter does not perform its own auth checks.
9. **Ordinary reply with no evidence.** Validator still allows the post (it's not type=evidence and lacks the `evidence` tag). The chip renders nothing on the node; the timeline contract renders no decoration. **This is the central doctrine assertion of EV-001** and is enforced by `__tests__/evidenceModel.test.ts → 'ordinary reply without evidence remains postable and renders no chip'`.
10. **Explicit Evidence post (`argument_type='evidence'`) with no source.** Server-side validator (`EVIDENCE_SOURCE_REQUIRED`) blocks the submit. EV-001 adds a test that mirrors this assertion to lock the behavior — i.e. "evidence-type post without artifacts must be blocked by `evaluateArgumentDraft`."
11. **Overrides set `broken` on an artifact that has full source+quote.** Allowed. Override wins. Chip + contract reflect `broken`. Used to model the admin-marks-chain-dead path.
12. **`addedByUserId` not provided.** Adapter requires it (TypeScript strict). Callers must pass `arguments.author_id`. There is no anonymous artifact path.
13. **Verdict-token sneak attempt** — a user pastes "this proves the liar wrong" into `label`. The label is truncated and rendered as-is in the chip — EV-001 does NOT sanitize user free-text content (that's a moderation concern, not the model's). The forbidden-token ban-list only covers the chip's *system-generated* strings (labels, helpers, accessibility suffixes).
14. **More than ~10 artifacts on one move.** Adapter still returns them all. Chip helper says "X receipts attached"; renderer is responsible for "+N more" UI. EV-001 does not cap the array.
15. **Label > 120 chars.** Truncated at 120 with an ellipsis (`…`). Adapter is deterministic.

## Test plan

All tests live in `__tests__/evidenceModel.test.ts` (single file keeps the EV-001 surface coherent; EV-002+ can fan out). Following the `test-discipline` skill (pure-model coverage + ban-list pattern):

### Shape + enum coverage

- `EvidenceArtifactKind` union has exactly the six documented values.
- `SourceChainStatus` union has exactly the five documented values (`no_source` mentioned in the card body maps to `unverified` — see §"Mapping note" below).
- `EvidenceRisk` union has exactly four values.
- `ALL_EVIDENCE_ARTIFACT_KINDS`, `ALL_SOURCE_CHAIN_STATUSES`, `ALL_EVIDENCE_RISKS` are frozen arrays and cover the unions exactly.
- An `EvidenceArtifact` object literal type-checks with every field at minimum example values.

### `deriveSourceChainStatus` — decision-table coverage

Eight tests, one per row in the table in §"Data model". Plus:
- Whitespace-only `url` / `sourceText` / `quote` is treated as absent.
- Setting all three with status overridden to `broken` returns `broken` (only via override path — `deriveSourceChainStatus` itself never returns `broken` / `primary_present`).

### `classifyEvidenceKind`

- Explicit `kind` always wins.
- Dataset allowlist (`data.gov`, `*.figshare.com`, `*.zenodo.org`) → `dataset`.
- Any other URL → `url`.
- `sourceText` only → `source_text`.
- `quote` only → `source_text`.
- All blank → not reachable from `buildEvidenceArtifacts` (dropped), but if called directly returns `manual_citation`.

### `buildEvidenceArtifacts` — adapter

- Empty input → `[]`.
- One URL attachment → one artifact, id `${argId}:evidence:0`, kind `url`, status `source_no_quote`.
- Two attachments, first all-blank → one artifact, id `${argId}:evidence:1` (stability promise).
- Three attachments → three artifacts, indices 0/1/2.
- Override sets `sourceChainStatus: 'broken'` on artifact 1 → shallow-merged.
- `label` truncation at 120 chars.
- Label fallback chain (explicit → hostname → sourceText prefix → "Attached evidence").
- Determinism: calling twice with same input returns deeply equal arrays.

### `summarizeArtifactsForReceiptChip`

- Empty → label "No receipts yet", tone `muted`, `invitesFollowup: true`, `showsSourceChainPressure: true`, `count: 0`.
- One `unverified` → label "Receipt attached".
- One `source_no_quote` → label "Source attached".
- One `source_and_quote` → label "Source and quote".
- One `primary_present` → label "Primary source".
- One `broken` → label "Source trail is weak", tone `attention`.
- Multiple artifacts → `kinds` array is the **unique-preserving** list in encounter order; `count` is the total. Worst status wins (`broken` > `unverified` > `source_no_quote` > `source_and_quote` > `primary_present`).

### `getTimelineEvidenceContract`

- `argumentType='evidence'`, artifacts empty → `rendersAsEvidenceNode: true`, `rendersSourceChainRing: true`, suffix mentions "Evidence node".
- `argumentType='rebuttal'`, artifacts non-empty `unverified` → `rendersAsEvidenceNode: false`, `rendersSourceChainRing: true`, suffix mentions "Has attached receipt".
- `argumentType='claim'`, artifacts empty → no decoration; suffix empty string.
- `argumentType='evidence'`, artifacts `source_and_quote` → `rendersAsEvidenceNode: true`, `rendersSourceChainRing: false`.
- `argumentType` undefined → treated as not-evidence (no node-decoration).

### Doctrine — missing evidence does NOT block ordinary replies

This is the EV-001 doctrine anchor (Acceptance criterion #4 from the issue body):

- Build the same with `argumentType='evidence'`, no evidence. Assert `result.allowPost === false` and at least one blocking flag with `flagCode === 'evidence_required'`.

## Mapping note — issue body status names vs internal model

The card body lists status names as `no_source | source_no_quote | source_and_quote | broken | primary_present`. The internal model uses `unverified | source_no_quote | source_and_quote | broken | primary_present`. The renaming reasoning:

- `no_source` describes the *absence* of an artifact, which the model treats as the empty-array case, not a per-artifact status.
- `unverified` describes the *presence* of an artifact whose chain hasn't been inspected.

A test asserts the empty-array path renders the chip's "No receipts yet" form, so the user-facing semantics promised by the issue body are preserved. The five operational per-artifact statuses (the ones EV-002 dispatches on) are: `unverified`, `source_no_quote`, `source_and_quote`, `broken`, `primary_present`. The issue body's intent — "EV-002 must read at minimum no_source / source_no_quote / source_and_quote / broken / primary_present" — is satisfied: `no_source` ↔ empty-artifacts (handled at the *array* level); the other four are first-class statuses.

This is the only deviation from the literal card text and it is intentional. If a future revision prefers `no_source` as a per-artifact value, replacing the enum is a single-file edit + test update.
