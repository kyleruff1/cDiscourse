**Card:** QOL-032 (Sidecar Rail · P1 · L · Release 6.6).
**Status:** Design.
**Depends on:** QOL-030 (the popout chassis).
**Companion:** [`one-box-interface-model.md`](../ux-storyboards/one-box-interface-model.md), [`QOL-030.md`](QOL-030.md).

---

## 0. How this design was built

Derived — **node stages in timeline mode** (§3) fix what Inspect emphasizes;
the **storyboard narratives** (§5) fix that the emphasis is right. Inspect is
the *understand* surface — it never authors anything.

## 1. Goal & scope

The **Inspect popout** is the read-only node / evidence / branch detail surface.
It converges:

- **SC-002** timeline node popover
- **SC-003** sidecar (not yet built — designed directly into the popout here)

In scope: the section set, stage-driven emphasis, the read-only contract, the
hand-off to Act, prev/next traversal. Out of scope: the chassis (QOL-030); the
semantic-flag *derivation* (RULE-001/003, META-001, MCP layer — consumed).

## 2. Inputs

QOL-030's chassis. Read-only: META-001 metadata ledger, LIFE-001 stage,
RULE-001/003 plain-language maps, the EV-001 evidence object, point-standing /
heat models. Inspect **renders** these; it derives nothing new and writes
nothing.

---

## 3. The Inspect popout in timeline mode

### 3.1 Anchoring & contract

Inspect anchors to the selected node (or its evidence pill / branch stub). It is
**strictly read-only** — no body editing, no posting. Editing is the box's job;
Inspect's "Suggested next move" *hands off* to the Act popout.

### 3.2 Sections (fixed set)

| # | Section | Source |
|---|---|---|
| 1 | What this move says | the argument body, plain |
| 2 | Why it matters | its relation to the parent (META-001) |
| 3 | What is unresolved | open axes / evidence debt (LIFE-001 + EV-003) |
| 4 | Where it sits | mainline / branch / tangent position (BR-001/004) |
| 5 | Suggested next move | a hand-off chip → opens the Act popout at that entry |
| 6 | Semantic flags | RULE-001/003 plain-language; **never raw `snake_case`** |
| E | Evidence detail | the EV-001 object — date, amount, note, status (QOL-023/024) |

### 3.3 Stage-driven emphasis

Every section is always present, but the section **pulled to the top and
expanded by default** follows the node's LIFE-001 stage. Emphasis only re-orders
— it never hides a section.

| Node stage | Inspect opens emphasizing |
|---|---|
| `open` · `answered` | §1 What this move says |
| `rebutted` · `clarified` | §3 What is unresolved |
| `source_requested` · `quote_requested` | §3 What is unresolved (the evidence debt) |
| `sourced` | §E Evidence detail |
| `narrowed` · `conceded` | §2 Why it matters (what the concession preserved) |
| `confirmed` · `synthesis_ready` | §5 Suggested next move (Synthesize) |
| `moved_on` · `ignored` · `exhausted` | §4 Where it sits (offer to branch) |
| `archived_or_resolved` | §1 + a settled banner |

## 4. Hand-off to Act

Inspect *understands*; Act *does*. §5 "Suggested next move" is the single
bridge: its chip is a `direct` affordance that closes Inspect and opens the
**Act popout** at the suggested entry. Inspect never opens the box itself — this
keeps the *understand* / *do* split clean and the popout count at three.

## 5. Narrative frames — Inspect in action

| Frame (scenario) | Node · stage | Inspect opens emphasizing |
|---|---|---|
| S1 F3 — A reviews B's response | node · `rebutted` | §3 — "Concession 2 disputed on **fact**; main refutation open" |
| S1 F5 — Observer C surveys before chiming in | node · `open` | §1 + §4 — what it says, where it sits (so C chimes in on the right node) |
| S1 F10 — the "door unlocked" line | node · `branch_recommended` | §4 — "introduces a new axis → consider a tangent branch" |
| S2 F3 — A inspects the evidence-backed node | node `sourced` · evidence object | §E — payment date, amount, note "practice space", **status: applicability disputed** |
| S2 F5 — after A asks for a source | node · `source_requested` | §3 — "Evidence requested — a source is owed here" |
| S2 F9 — at settlement | node · `archived_or_resolved` | §1 + settled banner — "March rent was paid; payment-label ambiguity" |

## 7. Edge cases

- **Node with no evidence** — §E renders "No evidence attached" (not hidden, so
  the section set is stable).
- **Own node** — Inspect is identical; read-only applies to everyone, so there
  is no own-node special case (unlike Act).
- **Archived room** — Inspect fully works (it is read-only); only the §5
  hand-off chip is disabled with the reason "this argument is settled".
- **Semantic flags absent** (MCP layer off) — §6 renders "No semantic flags";
  never a raw code.
- **prev/next at the ends** — wraps disabled at root / latest, consistent with
  IX-003 navigation.

## 8. Test plan

- The fixed section set is always all-present.
- Stage-driven emphasis — the §3.3 table: the right section is top+expanded for
  every stage; **no section is ever removed**.
- Narrative-frame tests — the §5 table.
- Read-only contract — Inspect exposes no posting / editing affordance (source
  scan + render test).
- Hand-off — §5 chip opens the Act popout at the named entry.
- No raw `snake_case` in any rendered string (RULE-001 ban-list).

## 9. Doctrine & safety self-check

- Strictly read-only — Inspect never writes, never posts, never opens the box.
- Plain language only — no internal codes (RULE-001/003).
- No verdict / winner / loser / truth copy; "what is unresolved" describes the
  *move*, never the person.
- Heat, if shown, is labelled activity — never correctness.
- Deterministic pure render over existing data — no AI, no network.

## 10. Dependencies & do not implement

- **Depends on:** QOL-030 (chassis); consumes META-001, LIFE-001, RULE-001/003,
  EV-001 read-only.
- **Do not implement here:** the chassis; the semantic-flag derivation; the
  evidence schema internals (QOL-023/024). Inspect *renders* them.
