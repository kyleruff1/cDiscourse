## Goal (one paragraph)

Render the **Referee Card** — one compact, three-zone synthesized card — on the **active node's** detail surface inside
the existing `CardDetailPanel` chassis. The card consumes REF-002's already-derived `DisagreementContract` and presents
**one coherent state per issue** (what this move is doing · what remains open · 2–3 next moves), never a stack of raw
families or flags. The doctrine that shapes every line: the deterministic Constitution engine
(`src/domain/constitution/engine.ts`, mirrored at `supabase/functions/_shared/constitution/`) is the **sole** submission
gate; the Referee Card is **advisory**, derives at render time **after** an argument is stored, **never** blocks a post,
**never** says a user is wrong, **never** accuses motive, and **always** offers a move (or collapses cleanly). Zone 1
**absorbs** the referee-banner meaning — there is exactly one banner surface; the card renders the banner's voice as
text + a non-color tone glyph, never a second `RefereeBannerView`. Raw family IDs / rawKeys / spans / numeric confidence
stay in Inspect (QOL-032). v1 is **derived-only**: no new data layer, no classifier call, no persistence, no Edge read
(cdiscourse-doctrine §1, §4, §5, §9, §10a).

## Data model

**No new data model.** REF-003 is a pure consumer of REF-002's shipped shapes:

- `DisagreementContract` (alias `OpenIssue`) — the derived issue, built by `buildOpenIssue(BuildOpenIssueInput)`.
- `RefereeCardViewModel` — the rendered shape, built by `openIssueToRefereeCard(issue)` / `buildRefereeCardViewModel(issue)`:
  - `zone1RelationLine: string` (banner headline **or** relation label)
  - `zone1ToneGlyph: 'star' | 'arrow' | 'branch' | null`
  - `zone2OpenTaskLine: string` (burden+axis **or** terminal-state line)
  - `zone3Moves: ReadonlyArray<MoveSuggestion>` (0–3 Act buttons; `MoveSuggestion = { actEntryId, label, accessibilityLabel, isRecoveryRoute, recoveredFromCode }`)
  - `refereeNoteSentence: string` (the full frozen-template sentence)
  - `accessibilityLabel: string` (one complete screen-reader sentence)

REF-003 adds **no** persisted table, **no** migration, **no** new union, **no** new field on any shape. The two new
locked plain-language strings the card introduces (the teaching empty state + the reused advisory caption) are
documented in **Copy** below and ban-list scanned.

## API / interface contracts

### `RefereeCardView` (the component)

```ts
// src/features/arguments/cardView/RefereeCardView.tsx
import type { DisagreementContract, MoveSuggestion } from '../../refereeLoop';

export interface RefereeCardViewProps {
  /** The REF-002 derived issue for the ACTIVE node. The component derives the
   *  RefereeCardViewModel via the shipped `buildRefereeCardViewModel`. */
  issue: DisagreementContract;
  /** v1 deep-link handler for a zone-3 move button. REF-004 swaps the surface
   *  implementation for full Act-popout routing; this leaf signature is stable. */
  onMove?: (move: MoveSuggestion) => void;
  /** Accepted for parity; the card never animates (reduce-motion is a no-op). */
  reduceMotionOverride?: boolean;
  testID?: string;
}
export function RefereeCardView(props: RefereeCardViewProps): React.ReactElement | null;
```

**Why the component takes the contract, not the pre-built view-model.** The component needs three things the
`RefereeCardViewModel` does not carry: `issue.refereeObservations.length` (to choose the teaching state vs the
observation-seeded zone 1), `issue.targetQuote` (the optional "point under dispute" anchor), and `issue.nextBestMoves`
(`= viewModel.zone3Moves`, carrying `actEntryId` for the handler payload). It calls the shipped
`buildRefereeCardViewModel(issue)` once in its body (pure, cheap, deterministic) to get every **rendered** string —
so REF-002 remains the **single** derivation owner and "consumes `RefereeCardViewModel` from the shipped model" is
satisfied literally (the model **is** that mapping function). The component renders **only** plain-language view-model
fields; `actEntryId` / `recoveredFromCode` / `issue.id` / mark `sourceCode`s are **never** rendered (the no-raw-codes
test pins this).

## Async + advisory (post never blocks; late observations re-derive)

- **The post path never waits on the card.** The Referee Card derives at **render time** from already-stored data; it is
  not in any submit path. Submitting an argument goes through the deterministic engine + `submit-argument`; the card is
  re-derived on the next render after the new row lands. The `RefereeCardView.test.tsx` includes a regression assertion
  that mounting/omitting the card never calls a submit function and that the composer submit path has no dependency on
  `refereeCardIssue`.
- **Pending / absent observation → neutral teaching state.** When the active node has **no** referee observation
  (`issue.refereeObservations.length === 0`: no banner, no machine observation) **and** the relation is the neutral
  `replies` fallback, zone 1 renders `'No referee notes yet on this move.'` — never an error, never a blocker, never
  "clean"/"no issues" (a verdict). Zones 2–3 still render their deterministic content (burden/axis + padded Act survivors).
- **Late-arriving observation rows re-derive the card.** The surface memoizes `refereeCardIssue` on the same dependency
  set as `activeCardDetail` — including `persistedObservationsByArgumentId`, `lifecycleMap`, `metadataLedger`,
  `evidenceDebts`, and the `refereeBanner` prop. When a classifier result (a new `persistedObservationsByArgumentId`
  entry or an updated `refereeBanner`) lands, the memo recomputes → `buildOpenIssue` re-derives → the card re-renders
  with the new zone 1 / zone 2 / zone 3. No imperative refresh, no subscription owned by the card.

## Sensitive gating (STOP-and-surface candidate — investigated, resolved)

The brief flags any sensitive-observation visibility ambiguity as a STOP that blocks automerge. Investigation:

- **The card renders only `RefereeCardViewModel` fields.** That view-model exposes zones 1–3, the referee-note sentence,
  and the accessibility sentence. It has **no** `userAllegations` field and **no** raw `machineObservations` field — so
  REF-003 renders **zero** allegations and **zero** raw marks. The surface cannot introduce its own observation-render
  path because the component never maps over `issue.machineObservations` / `issue.userAllegations` / `issue.refereeObservations[*].sourceCode`;
  it reads only the **length** of `issue.refereeObservations` (presence, for the teaching-state choice) and renders the
  model-summarized `zone1RelationLine`.
- **Family J is excluded at the model.** REF-002 re-asserts the `HUB_NON_PRODUCTION_FAMILIES` gate
  (`openIssueModel.ts:556-580`), so a `sensitive_composer` mark never reaches `refereeObservations`. The assembly passes
  `surface: 'timeline_node'` to `adaptAllSourcesForNode` — the **public**, highest-confidence-floor, already-J-gated
  surface used by `NodeLabelStrip` (`ArgumentGameSurface.tsx:1698`), not the composer surface.
- **Person-directed Allegations never reach the card.** They live in `issue.userAllegations` (visibility-gated by the
  model), which the card does not render at all (REF-005 owns the structured allegation surface).

**Conclusion: no visibility ambiguity, no bypass path → NOT a STOP.** The card's render surface is provably limited to
J-gated, allegation-free, plain-language view-model strings. The `refereeCardNoRawCodes` + sensitive-gating tests pin it.

## Accessibility (per `accessibility-targets`)

- **Targets ≥ 44×44.** Each zone-3 move is a `Pressable` with `minHeight: TOUCH_TARGET.minSizePx` + `hitSlop:
  TOUCH_TARGET.hitSlopAll` (the same tokens the inline `ActionsZone` uses, `CardDetailPanel.tsx:1114,666`).
- **Roles + labels + state per zone.** Move buttons: `accessibilityRole="button"`, `accessibilityLabel={move.accessibilityLabel}`
  (the verbose Act-entry label, never a key badge), `accessibilityState={{ disabled: false }}`. The note block:
  `accessibilityRole="text"` with `accessibilityLabel={vm.accessibilityLabel}` (one complete sentence describing the
  zone meaning). Caption + teaching state + anchor: `accessibilityRole="text"`.
- **Color-independence.** Meaning is carried by **shape + text**: the tone glyph is a geometric mark (`★`/`→`/`⑂`), the
  relation/burden/move words carry the state, the button has a visible border + text label. A grayscale snapshot stays
  legible (no color-only signal). The glyph is `accessibilityElementsHidden` — the tone is in the words.
- **Focus order matches reading order:** anchor → note (zone 1 + zone 2) → move buttons (left→right). The note block is a
  single SR stop; each button is its own stop.
- **Reduce-motion:** the card is fully static (no `Animated`, no transition). `reduceMotionOverride` is accepted for
  parity and is a no-op (mirrors `RefereeBannerView.tsx:127-130` and the panel's reduce-motion-safe contract). The
  `refereeCardA11y.test.tsx` asserts no animation and that the reduce-motion prop changes nothing.
- **No keyboard-badge surface.** The card introduces no keyboard shortcut, so no key badges and no `{platformOs,
  windowWidth}` badge gating apply.

## Edge cases (the implementer must handle)

- **No active node** → `refereeCardIssue` is `null` (surface returns null when `activeMessageId` is null) → the panel
  never mounts (no active card) → no card. No special case in the component.
- **`buildOpenIssue` returns null** (null room/target) → surface passes `refereeCard = null` → the slot renders nothing.
- **Empty room** → no nodes, no active card, no Referee Card.
- **Observer role** → `boardActRole === 'observer'`; `buildActPopout` survivors are watch/join/chime/view-qualifiers,
  none map to a `SuggestedMoveCode` → `zone3Moves` empty → zone 3 collapses; zones 1–2 still read the issue.
- **Own bubble** → `boardActRole === 'own_bubble'`; survivors are `view_qualifiers` + `request_deletion` → `zone3Moves`
  empty → zone 3 collapses (own-bubble safety: no edit/disagree/flag/score on your own move).
- **Root claim (`parentType === null`)** → relation defaults to `replies`; zone 3 may be `add_evidence` / `branch_tangent`
  only; zone 1 likely renders the teaching state when no banner.
- **No banner + no machine observation** → zone 1 renders the teaching state `'No referee notes yet on this move.'`.
- **Banner present** → zone 1 renders `vm.zone1RelationLine` (the banner headline) + the tone glyph.
- **Terminal state (`burden === 'none'`)** → zone 2 reads the terminal-state line ("Ready to synthesize." / "Conceded.")
  instead of an open-task line (REF-002 already does this in `zone2OpenTaskLine`).
- **Gate-removed suggestion** → REF-002 already substitutes the `branch_tangent` recovery route (or drops it); the card
  renders the recovery button's normal Act label ("Open a side issue") — never a rejection, never a disabled button.
- **Late classifier arrival / concurrent edits** → the memo re-derives on the next render; nothing blocks or delays the
  post in the meantime.
- **Family J mark in the persisted rows** → excluded by REF-002's gate; never reaches zone 1.
- **Heat / popularity / standing tries to influence the card → it doesn't.** No assembly field reads engagement, views,
  virality, or a standing band; `standingBand`/`evidentiaryRisk` are `null` and REF-002 never reads heat.
- **Verbatim `targetQuote` that contains underscores** (user typed `snake_case`-looking text) → rendered as user content
  in quote marks; the no-raw-codes test scans for **internal** code patterns (family ids, the `issue:` id prefix, mark
  rawKeys), not arbitrary user text, so user content is never flagged.

## Doctrine self-check

**Acceptance-gate invariant (verbatim, slate §2.1):**
*AI/MCP classifiers MUST NEVER be the submission acceptance gate. The deterministic rules engine
(`src/domain/constitution/engine.ts`, mirrored for the server at `supabase/functions/_shared/constitution/`) is the sole
gate. Classifiers run after an argument is stored. No path may block, reject, route, or delay an ordinary user post.*
→ Respected: the Referee Card derives at **render time**, after storage, from already-persisted data; it is never in any
submit path. Zone-3 moves are a **subset** of REF-002's engine+role-gate survivors — the card can never present a button
the engine would reject, and a structurally-invalid move is converted to a recovery route or dropped (never a
"click → rejected" path). The card emits no block/reject/route/delay.

- **cdiscourse-doctrine §1 (no truth labels; score never blocks).** Every rendered string is a frozen REF-001/REF-002
  label or one of two scanned locked constants; the 16 prohibited verdict/person tokens appear in this doc only as
  prohibitions and in the ban-list test. The card never says a user is wrong, never accuses motive, always gives a move
  (or collapses cleanly). Relation/burden/state describe the **move's** gameplay state, never truth.
- **§2 (heat = activity/friction).** No assembly field reads heat; "Moved on" / "Open" describe procedural state, not
  importance.
- **§3 (popularity is not evidence).** No assembly field reads engagement, views, virality, or a standing band
  (`standingBand`/`evidentiaryRisk` are `null`). Evidence debt is an obligation marker, never a verdict.
- **§4 (AI moderator limits).** No AI call from the production app; the card consumes already-persisted Observations and
  the already-derived banner; it never triggers classification, never assigns a truth value, never returns an
  authoritative flag.
- **§5 (engine is sacred).** The component + assembly helper import types only from the engine/domain; the assembly is
  pure (no React, no Supabase, no network, no `Date.now`); REF-002 (consumed unchanged) never re-validates the engine.
- **§9 (plain language).** No raw `snake_case` / family ID / rawKey reaches the card; the advisory caption is the imported
  constant; unknown codes were already suppressed upstream.
- **§10a (Observations vs Allegations).** The card renders Observation-derived zones only; it renders **zero**
  Allegations (the view-model has no allegation field). Sensitive Observations stay composer-only; Family J
  (`sensitive_composer`) is excluded by REF-002's re-asserted `HUB_NON_PRODUCTION_FAMILIES` gate; the assembly passes the
  public `timeline_node` surface.
- **§10 (v1 scope guards).** No voting/winner scoring, OAuth, push, public API, or search introduced.
- **Doctrine edge case — "what if heat tries to influence the axis/burden? — it doesn't":** no input is a
  heat/popularity/standing signal; the derivation (REF-002) is a pure function of structural move facts, and REF-003
  only renders it.
