# CDiscourse — User-Facing Terminology Audit

> Generated by `npm run ux:terminology:audit`
> (`scripts/ux/auditUserFacingTerminology.js`). Deterministic — re-run to refresh.
> Companion: `docs/ux-storyboards/terminology-and-copy-rules.md`.

CDiscourse is an **argument** product. Normal-user UI must not say
"game" and should not say "debate". This report scans **normal-user-
mode** app source (`src/`, `App.tsx`) for user-facing string literals
and JSX text that break those rules. It does **not** scan docs, tests,
scripts, the Supabase functions, or **admin / operator screens**
(`src/features/admin/`) — admin surfaces serve operators, not normal
users, and may use "debate" / "moderator" / technical terms. Database
table names like `debates` keep their internal names.

## How to fix

- Replace the term with the suggested replacement in the same row.
- If a match is a false positive (an internal identifier the scanner
  misread), add `ux-audit-ignore-line` as a trailing comment on that
  line, or add a justified entry to `ALLOWLIST` in the audit script.
- Database table names (`debates`) and internal code identifiers
  (`gameCopy`, `argumentGameSurface`) are intentionally out of scope.
