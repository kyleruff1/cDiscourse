## Summary

MCP-011 ships the pure-TypeScript foundation of the MCP semantic-referee stack:
the canonical Node-side packet contract (`semanticRefereeTypes.ts`), a strict
packet validator (`semanticRefereeValidator.ts`), a deterministic mock fixture
provider plus adversarial malformed-fixture set (`semanticRefereeFixtures.ts`),
and an order-normalized cache-key helper (`semanticRefereeCacheKey.ts`). The
implementation matches the approved design closely: all five §6.1 source files
and four §15 test files are present, the validator implements all six §8 stages
and every §10 rejection rule, the fixture provider matches §9, and the cache key
is the §11 superset. All four operator decisions in §18 are honored (zod@4 for
structure + pure-TS scanner; family-prefix `reasonCode`; `scoreHints` 0..3;
superset cache key; the fixture-contract doc ships). No new dependency, no
network code, no provider SDK, no env access, no wiring into any running
surface. Doctrine is respected throughout — `authoritative: false` is pinned,
verdict/person/secret/PII/chain-of-thought/raw-prompt/copy-smuggle shapes are
all rejected, and rejection messages never echo the offending raw value.
Verification is fully green and the test count rose 5813 → 5959. This is a
low-blast-radius, well-tested foundation card with nothing blocking.

## Confirmed expected false positives
The implementer documented these in design §16; the reviewer confirms each:
1. `semanticRefereeValidator.ts` `VERDICT_TOKENS` / `PERSON_LABEL_TOKENS` /
   `PERSON_LABEL_PHRASES` arrays — the ban-list the validator *enforces*, not
   user-facing strings.
2. `semanticRefereeFixtures.ts` and `semanticRefereeValidator.test.ts` defect
   strings (`troll`, `bad faith`, `winner_detected`) — synthetic adversarial
   values the validator must reject; assembled or minimal, no copied hostile
   text.
3. `semanticRefereeTypes.ts` `SemanticRejectionCode` union includes
   `verdict_token` / `person_label` / `secret_shape` — rejection-code *names*,
   validator internals.
4. The synthetic secret/handle/URL shapes in `semanticRefereeFixtures.ts` are
   assembled by concatenation at module load (`'sk-' + 'ant-' + 'X'.repeat(24)`,
   etc.) — no contiguous real-key-shaped literal; `semanticRefereeFixtures.test.ts`
   self-scans to prove it.
5. The two `grep` hits for `SERVICE_ROLE` / `ANTHROPIC_API_KEY` in `src/` are in
   pre-existing files (`src/lib/edgeFunctions.ts` doc comment,
   `src/features/devFixtures/argumentScenarioValidation.ts` redaction pattern) —
   not touched by this card.
