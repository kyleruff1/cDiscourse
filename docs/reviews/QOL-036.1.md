(`src/features/evidence/paymentEvidencePillState.ts`) that consumes COMP-001's

match the design §5. All four QOL-036 / QOL-037 / EV-003 / COMP-001 surfaces

`compositionLayer.ts` (zero logic change). No migration, no Edge Function,

console, no Supabase / network in deriver). Tests +59 / suites +1; full
suite 10452 / 415 passes. Typecheck + lint clean.

## Suggestions (non-blocking)

1. The §9.13 argument-order-independence test for the obligation axis
   shuffles the mutation array but the underlying within-axis last-wins
   resolution is sort-stable by `createdAt` first (not array index) — so
   the test is correct as written. A future enhancement could add a
   property-based test (e.g. with `fast-check`) over random
   chronology / classifier permutations to push the determinism story
   further. Strictly optional.
2. The `void layer2Classifier;` on line 490 (deriver) is a small
   strict-no-unused-vars accommodation that could be deleted if a
   `// eslint-disable-next-line @typescript-eslint/no-unused-vars` is
   preferred. Pure cosmetic — current style is fine.
3. The `CLASSIFIER_TIE_BREAK_ORDER` constant on line 214 documents its
   logic in a comment but is not exported. If a future card needs the
   same ordering (e.g. for an admin debugger that mirrors layer-2
   chronology), it could be promoted to a `readonly` export. Not needed
   today.
