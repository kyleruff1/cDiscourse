## Summary

REF-005 replaces two (in practice four) loose one-tap "flag/accuse" affordances with a
**procedural allegation object** — a moderation concern expressible only as (a) a required
target quote, (b) a bounded 6-value concern type, and (c) a bounded 6-value requested
remedy, never as free text on a person. The new pure-TS model
(`requestReviewModel.ts`), the 3-step presentational composer (`RequestReviewComposer.tsx`),
and the `ArgumentGameSurface` rebind are all clean, well-tested, and doctrine-conformant.
The persistence boundary (Option (b): composer-local, no `supabase/**`/Edge/migration/RLS)
is honored exactly; REF-005B (GATE-C) is correctly named as the follow-up. The single §8
deferral (the `actPopoutModel.ts` relabel blocked by the UX-001.5 zero-diff guard) was
handled **correctly** — the implementer refused to loosen a failing guard (a never-self-approve
action) and reverted the file byte-identical while delivering the full behavioral rebind. The
only residual is a cosmetic label inconsistency on two secondary entry points; it carries no
doctrine weight and is surfaced below as an explicit operator question. No blockers.
