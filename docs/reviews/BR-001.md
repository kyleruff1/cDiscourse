Tests, typecheck, and lint all green at 2509/97 passing.

## Three reported behavior deltas — doctrine-correct AND test-supported

1. **`isDetached === true` always returns `'detached'`** — Doctrine-correct: detached means "parent missing from loaded set"; family / tags should not override that. Asserted in `__tests__/railSegmentModel.test.ts:398-417` and `__tests__/branchTopologyModel.test.ts:355-364, 443-459`.
2. **Non-detached `siblingIndex 0` with `branch_this_off` / `tangent_or_joke` tag → `'kink_start'`** — Doctrine-correct: explicit user/moderator-tagged tangents are the one place the topology can override "first child on lane" because the user / moderator has *labeled* the move as off-branch. Asserted in `__tests__/railSegmentModel.test.ts:427-431` and `__tests__/branchTopologyModel.test.ts:391-400`.
3. **Non-detached `siblingIndex ≥ 1` → `'kink_start'`** — Doctrine-correct: this is the headline AC ("additional children branch up/down"). Doctrine: this still says "leaves the mainline geometrically", never "off-topic" or "wrong". Asserted in `__tests__/railSegmentModel.test.ts:434-438` and `__tests__/branchTopologyModel.test.ts:417-441`.

## Suggestions (non-blocking)

1. **`buildBranchKindMap` pass 2a uses a fixed-point loop with a `sweeps < args.edges.length + 1` cap.** In practice the loop converges in 1–2 sweeps (the design explicitly notes this), but the `O(n²)` worst-case cap is wider than the design's "O(depth) sweeps". A follow-up could index edges by `fromMessageId` once and walk the children-of-tangents in a single ordered pass keyed by depth. Not blocking — current performance test at depth ~25 stays well under 50 ms.
2. **`looksLikeInternalCode` ban-list import is `from '../src/features/arguments/gameCopy'`** in the model tests. If a future card moves that helper, the BR-001 tests need to follow. Consider re-exporting from `branchTopologyModel.ts` or pinning it via a stable barrel.
3. **`BranchCollapseStub.tsx` testing pattern is "test the pure helpers, not the rendered tree".** This follows the existing EV-002 `ReceiptChip.test.tsx` precedent and stays consistent with the repo's "no RN renderer in jest" discipline — keeping it. If a future card adds @testing-library/react-native, the stub becomes trivially testable through that. Logged for IX-003 / PR-001's a11y conformance pass.
