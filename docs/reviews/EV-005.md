## Summary

EV-005 ships the full-scope card in one pass: a pure-TS evidence-annotation
model (18 source/record-descriptive kinds, derived status chip, one-level depth
cap), a read-only annotation stream + status chip inside the EV-002
`SourceChainPopover`, the interactive "Add an annotation" picker
(`AddAnnotationSheet`), the `annotate-evidence` Edge Function write path with a
Deno-side eligibility mirror, and the Timeline/Cards receipt-count reflection.
All four verification commands pass (typecheck, lint, 6901 tests / 266 suites,
skills:validate). Safety scans are clean — service-role appears only inside the
Edge Function, never in `src/**`/`app/**`; no secrets; no migration; no client
AI; no direct `arguments` write from the client. Doctrine holds: the ban-list
test genuinely scans every app-authored string, the model imports nothing at all
(so an annotation has no structural path to a `PointStandingDelta`), and the
depth cap is proven with a true depth-2 record. Both known implementer
deviations were verified benign. Footprint matches the full-scope design. One
non-blocking nit: a stale comment reference. Ready to push and PR.

## Known deviations — both verified benign

1. **`evidenceAnnotationApi` not re-exported from the evidence barrel `index.ts`.**
   Confirmed sound. The barrel is consumed by EV-001's pure-model tests;
   re-exporting the wrapper would pull `lib/supabase` + AsyncStorage into those
   tests. `metadata/pointTagsApi.ts` is likewise absent from its barrel — same
   established pattern. `index.ts` carries an explicit comment explaining this,
   and `TimelineNodePopover.tsx` imports `addEvidenceAnnotation` directly from
   `'../evidence/evidenceAnnotationApi'`. No consumer breaks; typecheck + the
   full suite pass.

2. **UI test files use pure-helper + source-scan discipline, not runtime RTL.**
   Confirmed coverage is real, not hollow. The eligibility-gating tests execute
   the real `eligibleAnnotationKinds` pure model; copy assertions check real
   exported constants; the chip tests execute real pure label helpers. The
   source-scan portion is verified against the actual component source — the
   `AddAnnotationSheet.tsx` / `EvidenceAnnotationChip.tsx` sources genuinely
   contain the asserted a11y contract (radio rows, `accessibilityViewIsModal`,
   Escape/scrim/back close, `reduceMotion`, ≥44 targets, polite live region).
   This matches the repo's established discipline (`ReceiptChip.test.tsx`,
   `ix003KeyboardA11yWiring.test.ts`) given react-test-renderer is pinned away
   from @testing-library's peer.
