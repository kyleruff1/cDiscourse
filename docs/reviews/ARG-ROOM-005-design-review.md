## Actionable comments
1. **[should] Tighten `classifyJoinOutcome` so a bare `23514` without the token is NOT `room_full`.** The spec says "message includes 'room_capacity_reached' (primary) **and/or** SQLSTATE '23514'." `23514` is generic `check_violation`; an unrelated check on `debate_participants` returning `23514` would be misclassified as full and silently route the user to observe. Make the `'room_capacity_reached'` message token authoritative; treat `23514`-without-token as `unavailable`/`error`. Add a negative test: `classifyJoinOutcome({code:'23514', message:'some other check'})` must not return `room_full`.
2. **[should] Add a no-differential-signal doctrine test.** Assert that for a public viewer (`knownReservedInviteCount=0`), a room with a hidden reserved invite and a room with one more genuinely-open seat produce **identical** `SeatAvailability`/view-model output — locking the privacy property that the reserved seat is invisible at the preview layer (complements the existing "refusal copy is identical" claim).
3. **[nit] Be deliberate about the public over-count preview.** A public viewer of a (creator + 1 hidden reserved + 3 open) room will see "4 open seats" and one claimer will be bounced to observe. This is the acknowledged, correct privacy cost — keep it, but add a one-line code comment at the preview site citing the reserved-RLS-hidden rationale so a future maintainer doesn't "fix" it into an enumeration leak.
4. **[nit] Specify the fallback participant read as a bounded count.** Where a mount lacks participants, use a `head:true`/count read (not a full-row fetch) so the standard-gated client read stays cheap; note it reads an existing RLS-readable table (no new server object).
5. **[nit] File the two deferred server hardenings as their own cards.** (a) the optional 002-follow-up dropping `reserved=` from the trigger DETAIL (GATE-C, Open Q #5); (b) a BEFORE-UPDATE branch on `enforce_room_capacity` so a future observer→active promotion can't bypass the cap (GATE-C, Open Q #4). Both are out of 005 scope but should be tracked so they aren't lost.
6. **[nit] Comment `SEAT_CLAIM_COPY` as 007-owned.** Land it minimal now (correct call — keeps 005 testable/ban-list-covered) with a pointer that ARG-ROOM-007 consolidates the final vocabulary, to flag the expected rename churn.
