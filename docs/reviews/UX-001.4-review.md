the load-bearing handoff for UX-001.5 is in good shape. Typecheck + lint

## Findings

None blocking.

**Observations (non-blocking):**

1. **`DebateDetailHeader.tsx` cleaner than design.** The brief permitted
   touching the strip if necessary. The implementer chose to mount the
   trigger row INSIDE `ArgumentGameSurface.tsx` (between Timeline body
   and side action rail), leaving `DebateDetailHeader.tsx` byte-equal.
   This is a strictly cleaner outcome — the UX-001.2 strip contract is
   not just preserved, it is untouched. Documented in Phase 4 framing
   under "Deferred questions" §"Strip-level trigger placement".

2. **Composer focus check uses a sensible approximation.** The board
   keyboard handler at `ArgumentGameSurface.tsx:1099-1106` reads
   `document.activeElement` and treats INPUT / TEXTAREA / contenteditable
   as composer-focused. The implementer documented why this works (the
   composer's TextInput is the only such element rendered while the
   room is mounted) and the test suite asserts the resulting effect.
   For UX-001.5, if a new TextInput is added inside the surface
   (unlikely per current scope), the approximation may need refinement.

3. **Inspect's `evidence_detail` section is empty in v1.** The
   `buildInspectContent` JSDoc explicitly notes this is a v1 known
   limitation (the sidecar view-model does not currently carry the
   EV-001 evidence-object detail; the evidence contract is on the
   timelineMap). Inspect's §7 fallback ("No evidence attached") renders.
   This is design-intentional — wiring evidence through is a follow-up
   card.

4. **A `note` for UX-001.5.** The Phase 4 framing section's "Inspect
   surface contract" subsection is unusually thorough — 50+ lines naming
   `InspectSectionContent`'s 7 sections explicitly, naming the
   `buildInspectContent` helper, explicitly calling out that UX-001.5A
   extends the `flags` section additively. This is exactly the
   load-bearing handoff the brief required.
