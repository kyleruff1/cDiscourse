## Summary

AN-003 ships a pure-TS, dev/debug-only tree-playability diagnostics deriver
(`src/features/analytics/treePlayabilityDiagnostics.ts`, 731 lines) plus the §5
snapshot generator (`scripts/diagnostics/buildTreePlayabilitySnapshot.ts`) and
its standalone CommonJS `tsconfig.diagnostics.json`. The model consumes the
already-built `PointLifecycleMap`, `ArgumentTimelineMapModel`, GAME-001
advisories, EV-001 contracts and SC-004 dock models, and re-derives none of
them. It produces verdict-free structural counts (`playabilityClass`,
unresolved/stale fractions, branch depth, evidence-debt concentration,
no-actionable-suggestion count, per-state breakdown, FNV-1a fingerprint) and a
dev-only Markdown snapshot written exclusively to gitignored `artifacts/`. The
implementation matches the revised design precisely, all card-vs-reality
adaptations (D1–D7) are handled correctly, the four §5 deviations are each
sound and well-documented, doctrine is clean, and the verification battery
passes (4287/4287 tests, 146/146 suites — a +29/+1 delta vs the 4258/145
baseline). No production app code, no migration, no Edge Function, no service
role, no AI, no new dependency. Nothing concerning remains. Approve.

### §5 implementer deviations — all four verified sound

1. **Standalone `tsconfig.diagnostics.json` (does not `extends` root tsconfig)**
   — correct. The root tsconfig inherits `expo/tsconfig.base`, which carries a
   `customConditions: ["react-native"]` that TypeScript only accepts under a
   bundler/node16 resolver, incompatible with the `moduleResolution: node` the
   bare-`node` run needs. The standalone config keeps `"strict": true` and adds
   `esModuleInterop`/`forceConsistentCasingInFileNames`/`skipLibCheck` — type
   safety for the script is not weakened (strict mode preserved; `skipLibCheck`
   only skips `.d.ts` checking, standard for a build-only config).
2. **`files` (entry only) instead of `include: src/**`** — sound. `tsc` follows
   the actual import closure from the single entry; listing `src/**` would
   force-compile barrels that re-export `.tsx` components and pull DOM/Supabase
   code into the compile. `"rootDir": "."` pins the emit layout so the
   `package.json` `node` paths are exact.
3. **Inline `buildArtifactsByMessageId` instead of importing EV-002's helper**
   — sound and faithful. Compared line-for-line against
   `src/features/arguments/argumentGameSurfaceEvidence.ts`: same
   `buildEvidenceArtifacts` call, same `?? []` guard, same
   `authorId || 'unknown'` fallback. The only difference is `Map` vs `Record`
   return shape — required because `buildPointLifecycleMap` expects a Map. It
   re-derives nothing (calls the shipped EV-001 builder) and correctly avoids
   the evidence barrel that re-exports `.tsx`.
4. **`eslint.config.js` `artifacts/**` ignore** — correct and narrowly scoped.
   Added alongside the existing `node_modules/**`, `.expo/**`, `dist/**`,
   `web-build/**` ignores; `artifacts/` is already gitignored throwaway build
   output. ESLint must not lint generated emit.
