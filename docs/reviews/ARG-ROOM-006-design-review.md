## Gating + doctrine + security
- **Gating verdict is correct** (see header). The (a)–(f)/(g) split is real and load-bearing; (g)'s GATE-C is justified by the verified `config.toml` registration.
- **Item (c) needs NO RLS — verified.** The QOL-039 debates SELECT policy (189–198) already withholds private rooms from non-members at the DB, so `listDebates` (which adds no WHERE) never returns them. `resolveRoomDeepLinkAccess` is pure client logic over the already-RLS-filtered id set; a requested id absent from that set yields `unavailable`, which is **identical** for private-no-access and nonexistent — the no-enumeration guarantee by construction. The design neither weakens nor re-implements RLS.
- **No enumeration.** Deep-link unavailable copy is cause-neutral; item (g)'s 409 names no one and uses `head:true` + count-by-`debate_id`/`status` only (no email/token, no per-email oracle); the mismatch panel echoes only the viewer's own email (561–563 backend + 270–279 UI).
- **No verdict copy.** `ROOM_ACCESS_COPY` strings are access/seat facts ("Seats are full — you can still observe"); "full" is a seat fact, observers stay uncapped (`canObserve` always true on a readable room). Ban-list enforced via `_forbiddenRoomAccessTokens()` + the existing gameCopy scan.
- **No raw token / no service-role in client.** Pure client logic over RLS-filtered data; (g) stays inside the Edge Function.
- **Plain language.** The new invite-error key routes through `plainLanguageForInviteError`; `ROOM_ACCESS_COPY` is authored copy (not codes) scanned by the ban-list — consistent with the `ROOM_VISIBILITY_COPY` pattern, so it correctly does not need `toPlainLanguage` routing.
