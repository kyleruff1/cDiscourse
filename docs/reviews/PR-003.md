## Summary

PR-003 ships the first storage-backed upload pipeline in the repo: `profile-avatars` public-read bucket (2 MiB cap, JPG/PNG/WebP), four new `profiles` columns (`avatar_path`, `avatar_thumb_path`, `avatar_updated_at`, `avatar_moderation_status`), a narrowed UPDATE policy that freezes avatar columns against client-JWT writes via a same-table OLD subselect, and a three-action `upload-avatar` Edge Function (upload / remove / read_url_for_user) that uses `imagescript@1.2.17` for server-side decode + resize + WebP re-encode (EXIF stripped by construction). The client surface is `AvatarUploadSection.tsx` mounted inside the existing `AccountScreen` card with optimistic local-URI rendering and a two-tap remove pattern. Every acceptance criterion is enforced at the data layer, not the UI layer. The Profile epic framing section in `docs/core/current-status.md` is comprehensive and PR-004-ready (each Q1-Q6 pattern has a file-anchored entry plus a "PR-004 consumes:" handoff). Verification battery and protected-boundary checks all clean.
