## Summary

GAL-001 expands the Stage 6.4 Conversation Gallery's 6-section grouping into a 10-lane "scrum board" where each lane invites a specific kind of move. The change is a pure-model + chip-row UI swap — no new dependency, no Supabase touch, no AI call, no migration. Lane derivation in `classifyCardToSection` is total, deterministic, side-effect-free, and reads only structural / activity / lifecycle signals (no popularity / engagement / virality, doctrine §3). Every lane label, helperLine, and emptyCopy string is plain-language and ban-list-clean. The Stage 6.4 `easy_first_move` code is renamed to `quiet_beginner_rooms` (semantics unchanged); the Stage 6.4 `hot_unresolved` code is retired (cards split between `jump_in` and `logic_traps`). The bucket chip row is removed from the gallery UI per the resolved Q1 in the implementer brief — `BUCKET_DEFINITIONS` and `ConversationBucket` survive as internal classification feeding lane derivation. Test count goes from 3762 → 3913 passing (+151 tests across 3 new suites), all pinning real doctrine invariants. Typecheck + lint clean. The 19 pre-existing failing tests are exclusively operator-gated xAI / Anthropic suites under `scripts/bot-fixtures/` (env_file_missing) and are unrelated.

## Behavior-change framing

For future readers — what changed at runtime and why it is safe:

1. **`easy_first_move` → `quiet_beginner_rooms` (rename, same classification rule).** Stage 6.4 callsites that asserted the old name (one test) are updated in lockstep. The classification predicate (`bucket === 'pedantic_plain'` ∪ `heatLevel === 'cold'` fallback) is preserved.
2. **`hot_unresolved` retired (cards now split).** The Stage 6.4 lane fanned three input shapes into one bucket; GAL-001 splits them by structure:
   - `bucket === 'hot_now'` → `jump_in` (active back-and-forth is an action lane).
   - `bucket === 'unresolved_deep_chain'` → `logic_traps` (specific move-structure signal).
   - `heatLevel === 'overheated'` → `jump_in` fallback (active activity).

   No production runtime code reads the string `hot_unresolved`; only one test + docs. Doc/test references are updated.
3. **Six new lanes**: `evidence_needed` (from `evidence_fight` bucket), `definition_fights` (from `definition_scope_fight` bucket), `logic_traps` (from `exhausted` lifecycle ∪ `unresolved_deep_chain` bucket ∪ `deep_existing_clash` entryOpportunity), `tangents_branches` (from `branch_recommended` lifecycle), `almost_synthesis` (from `synthesis_ready` ∪ `narrowed` ∪ `conceded` ∪ `confirmed` lifecycle), `quiet_beginner_rooms` (the catch-all fallback).
4. **UI change**: the previous bucket-chip row is replaced by a lane-chip row (single-select, observer-safe, toggle-to-clear). Sort + search behaviour preserved.
5. **`heatModel.ts` `EntryOpportunity` union still includes `easy_first_move`.** This is SW-002's `EntryOpportunity` token (an orthogonal vocabulary read by GAL-002 hint deriver, never rendered as user copy). Intentional; not a leak.

## Nits / follow-ups (non-blocking)

1. **Legacy `GALLERY_SECTIONS` alias** (`src/features/arguments/gameCopy.ts:424`). The implementer kept `export const GALLERY_SECTIONS = GALLERY_SECTION_DEFINITIONS;` for one-release back-compat so `seamlessConversationEntry.test.ts` keeps the historical import name. The alias is currently consumed only by that one test (which now asserts the 10-id catalogue), so the alias has done its job. **Follow-up:** in the next release that includes a GAL-related touch, drop the alias and rename the test import to `GALLERY_SECTION_DEFINITIONS`. Add a brief retirement note in `docs/seamless-conversation-entry.md`.

2. **Style key names `bucketRow` / `bucketChip*` survive in the stylesheet** (`ConversationGalleryScreen.tsx:565-570`) even though the row no longer contains bucket chips. The visual primitive carries forward; only the consumer renamed. **Follow-up (cosmetic):** rename to `chipRow` / `filterChip*` next time anyone is in this file. Pure rename, zero-functional.

3. **`groupGalleryCardsBySection(paged.page)` groups within the current pagination slice** (`ConversationGalleryScreen.tsx:327`). If a lane has cards only on later pages, it disappears from the current page's all-lanes view. This is consistent with the existing pagination model (sort then page then group), and matches the design — but if user feedback says "the gallery feels inconsistent across pages," a future card could group first then paginate within each lane. Not a defect today.

4. **`heatModel.ts` still uses `'easy_first_move'`** (SW-002 `EntryOpportunity`). Intentional and isolated — the `EntryOpportunity` union is a separate vocabulary that GAL-002's hint deriver reads. Not user-facing, never rendered. Flagged here so future readers do not mistake it for a stale rename gap.

5. **Empty-state for the "all lanes" mode when every lane is empty.** When `paged.page.length === 0`, the screen renders `<EmptyState title={emptyTitleForLane('all')} body={emptyCopyForLane('all')} />` — strings "No rooms found" / "Try a different lane or clear search." That is good. Edge case: if the lane filter is `'all'` and search is empty and there are no rooms at all (no `debates` at all), the screen still falls through to the generic empty state. Consider a more inviting first-run state in a future PR if telemetry shows new-user confusion.
