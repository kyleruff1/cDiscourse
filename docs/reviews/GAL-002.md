## Summary

GAL-002 ships exactly what the design promised: a pure-TS rename and refactor of Stage 6.4's `deriveConversationEntryHint` into `deriveGalleryEntryHint`. The new deriver routes plain-language copy through RULE-003 (`getLifecycleUx().label/.helperLine`), dispatches primarily on LIFE-001's `rootClusterLifecycleState`, falls back deterministically to the Stage 6.4 bucket parity, applies SW-002's `entryOpportunity` as a tie-breaker only on `watch_first` paths, and emits a single typed `GalleryEntryHint` carrying `activate` / `code` / `verbPhrase` / `helperLine` / `presetKey` / `dockAction`. Render-layer change is a one-style-alias addition (`microMomentHelper`) plus an `accessibilityLabel` that combines both lines — the minimum visual surface the design described. Doctrine is respected end-to-end: no truth labels, no popularity inputs, no AI calls, no service-role, no DB migration, no `.env` touch. Tests grew from 3665 → 3700 (+35) with no removals; the 19 baseline failures are pre-existing AI-adapter env-detection tests that fail in a clean worktree (no `.env.engagement-intelligence` present) and have zero overlap with the GAL-002 diff. Approve and push.

## Specific positives worth noting

1. **The freshness test (§2) is unusually well-engineered.** It anchors the deriver region with two literal header strings (`// ── GAL-002 — Gallery entry-hint deriver` and `// ── End GAL-002 region`) and scans only inside that slice for `^let` / `new Map(` / `new WeakMap(` / `new Set(`. Both markers are present in the source. The single in-region `let choice: HintChoice` is inside the function body and is correctly excluded by the `^let` line-start regex.
2. **Helper-line specificity for lifecycle-derived hints (`helperLineForHint`).** When the resolved lifecycle is `quote_requested` / `exhausted` / `branch_recommended` / `rebutted` / `conceded` / `confirmed` / `narrowed` / `ignored_by_both`, the deriver pulls the RULE-003 helper for that specific state instead of the generic mapped-code helper — so the surface copy stays anchored to what the cluster actually shows. The design called this out and the implementation honours it.
3. **The `applyEntryOpportunity` tie-breaker is correctly bounded.** It only promotes `watch_first` → `narrow` / `join_when_ready` (never the reverse), and the test `entryOpportunity never overrides a non-watch_first choice` pins this. This preserves doctrine §3 (SW-002's `entryOpportunity` is activity-derived, not popularity-derived, but the tie-breaker still defers to lifecycle/bucket as the primary signal).
4. **The render-layer accessibility wiring concatenates the two lines into a single utterance for screen readers** rather than letting them be announced as two disconnected views — better SR UX with no extra widget.

## Suggestions (non-blocking)

1. **Capitalisation consistency in the helper-line table.** RULE-003's `helperLine` strings start with a capital letter ("Source requested", "An admin or synthesis closed this cluster."). The 3 META helpers also use sentence case. The combined render `verbPhrase. helperLine` results in two capitalised fragments separated by a period inside the `accessibilityLabel` — fine, just worth confirming on a real screen reader that the period-separator reads cleanly.
2. **The bucket-fallback for `gaining_heat` produces `watch_first`** while the lifecycle path for `rebutted` produces `narrow`. A loader that plumbs lifecycle for some cards but not others will get visibly different first-suggested moves for what visually looks like the same heat state. This is design-correct (lifecycle is a stronger signal) but worth a one-line note in `docs/conversation-gallery-ux.md` once GAL-002 lands so the operator isn't surprised.
3. **Future PRESET-001 / GAL-003 follow-up:** the design notes these are the proper home for new `QuickActionLabel` presets and richer hint chips on the gallery card itself. The current PR leaves no debt on this — the deriver's `presetKey: null` for meta hints is the correct contract until the composer side learns new presets.

None of the above is a blocker; the card ships as designed.
