## Suggestions (non-blocking)

1. **Phone 0-slack maintenance concern.** The Phone band's offset arithmetic equals 128/128 px. The brief permits ≤ 128 and the test correctly uses `toBeLessThanOrEqual`. However, a future maintenance pixel change (e.g., accessibility-scale text expansion that grows the strip's `rowMinHeight`, an iOS SafeArea inset, a dev-environment-banner becoming production, or the AppHeader's measured height being slightly higher than the design token under RN-web's actual render) could breach the cap. Two mitigation paths if observed:
   - Tighten `containerPaddingVertical` from 4 to 3 on phone (recovers 2 px slack).
   - Tighten `rowMinHeight` from 36 to 34 on phone (recovers 2 px slack), provided `hitSlop` is increased correspondingly to preserve the 44×44 hit target.
   No action required at merge; the offset test would catch any future regression at PR time. This is recorded as a follow-up concern, not a blocker.

2. **Phone band 1366×768 stress check.** The brief specifies 1366×768 as a critical hard-blocker viewport, but the offset acceptance test computes the arithmetic against the band tokens (wide for 1366×768) rather than rendering the full chain at the explicit pixel dimensions. The arithmetic verifies the upper bound; a future render-test that mounts the full room chain inside a `useWindowDimensions` mock at the 4 hard-blocker viewports would provide additional defense-in-depth. Optional UX-001.6 cross-device QA scope.

3. **Mini-map collapsed-header path.** The design's Q10 Option (b) was chosen — the mini-map's collapsed header is accepted as part of slack rather than moved into the overlay. The arithmetic table accommodates this; the mini-map is not rendered in the default state (it has its own length threshold). If a long-room rendering surfaces measurement drift, the mini-map header may need to move into the overlay container per Q10 Option (a). The mitigation is documented in the design § Risks-and-trade-offs and does not require pre-merge action.

4. **microMoment exemption acknowledgement.** The operator's Option 1 override (transient + exempt + dismiss-on-first-meaningful-Timeline-interaction) is applied at the ArgumentGameSurface inline path. The implementer's commit message and the Phase 2 framing both document the QOL-040.3 contract update. UX-001.3's designer should not re-introduce a persistent microMoment variant; this is captured in the framing.
