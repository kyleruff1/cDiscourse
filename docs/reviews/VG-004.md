## Summary

VG-004 ships six render-only refinements to the horizontal argument board (`ArgumentTimelineMap`): a node-level active-path glow, a selected-node halo, an evidence receipt inner mark, a branch-stub glyph refinement, a tone tint on active-path nodes, and density-aware inter-node spacing. All new visual logic is extracted into a new pure-TS model (`timelineNodeVisualModel.ts`) with a dedicated 47-test suite; the component changes are thin render layers driven by the pure helper. The implementation respects the design with one operator-approved divergence (Q1, see below) and one design-documented deviation (the `accessibilityFragment` "selected" → "selected for actions" phrasing). Typecheck, lint, and the full 4258-test / 145-suite battery all pass. No secrets, no doctrine violations, no new dependency, no Supabase/Edge Function/AI path. Ready to push.

### `accessibilityFragment` "selected" → "selected for actions" — verified sound
`looksLikeInternalCode` (`gameCopy.ts:280-289`) matches `/^[a-z][a-z0-9_]{4,}$/` — a bare lowercase 5+-char single token with no spaces. A bare `"selected"` (8 chars, lowercase, no space) would trip it. The implementer's fix to the multi-word `"selected for actions"` contains spaces and matches none of the four regexes. Test group 7 (`argumentNodeVisualVG004.test.ts:437-441`) sweeps `looksLikeInternalCode(fragment) === false` across the full cross-product of inputs and genuinely passes (full suite green). Deviation is sound and documented in the design's deviation note.
