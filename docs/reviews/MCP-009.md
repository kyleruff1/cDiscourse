## Suggestions (non-blocking)

1. The future implementation card should add a **type-parity test** between
   `supabase/functions/_shared/semanticReferee/types.ts` (Deno) and
   `src/features/semanticReferee/types.ts` (Node). The design names the drift risk in ¬ß"Risks" but
   only calls a parity test "advisable" ‚Äî recommend it be made a required test, since the
   language-processing scaffold already lives with the same dual-mirror and a contract change must
   update both.
2. ¬ß"Edge cases" leaves the empty-`moveBodyRedacted` handling as an either/or (schema-reject vs
   mock-returns-all-zero) with a recommendation to reject. The future card should lock the decision
   so the schema and the mock provider agree; a half-resolved choice is a small footgun for the
   implementer.
3. The doc's `ClassifyMoveOutcome` lists `reason: 'not_implemented'` as a disabled reason. The
   future card should confirm `not_implemented` is reachable only for an explicitly unwired provider
   slot (mirroring `process-language-draft`'s `openai` branch), so it never leaks for a genuine
   misconfiguration that should read `not_configured`.
