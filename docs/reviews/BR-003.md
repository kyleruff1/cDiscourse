## Summary
BR-003 ships a deterministic, pure-TS tangent-routing model (`tangentRoutingModel.ts`)
that assesses how strongly a composer draft reads as a structural redirect away
from the point it replies to, and surfaces the result as exactly ONE advisory
inside RULE-004's existing `PreSendReviewSheet`. The implementation matches the
design doc precisely: the RULE-004 model is extended by exactly one `AdvisoryKind`
member (`tangent_redirect`), one `ADVISORY_DEFINITIONS` entry, one optional
`tangentContext` input field, and one derivation step (step 9), with the de-dup
rule against `asks_new_question` implemented as specified. `PreSendReviewSheet.tsx`
is not touched â€” the new advisory renders through the sheet's existing generic
advisory loop. The assessment is structural-only (every `RedirectReason` describes
the move's relation to its parent, never the person), advisory-only (zero new
blocking rules), deterministic (no AI, no network, no body-keyword matching, no
heat/popularity read). The file footprint is exactly the 12 files expected. All
verification passes; the only test failures are the documented pre-existing
environmental failures in the xai/aiDrivenBot bot-fixture suites. No concerns
remain.
