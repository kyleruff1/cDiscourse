## Summary
QOL-026 fixes the fully-broken Admin → All Arguments tab, which failed with
`column arguments.selected_tag_codes does not exist`. The loader at
`src/features/admin/adminArgumentsApi.ts` requested two columns that have never
existed on `public.arguments`. The implementation applies the two treatments the
design specifies: `selected_tag_codes` is replaced by a correct PostgREST nested
embed `argument_tags(tag_code)` flattened through a new exported pure helper
`asTagCodes`, and `attached_evidence` (plus `hasEvidence` and the evidence badge)
is dropped entirely with no `hasEvidence: false` hardcode. The change is a
SELECT-only client read on the shared anon RLS-scoped client — no migration, no
Edge Function, no service-role, no AI path. Diff is tight (3 production files,
2 test files, 1 doc), every claimed scan is clean, typecheck/lint/tests pass.
No concerns remain.
