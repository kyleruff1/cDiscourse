## Summary

QOL-042 ships the cross-room "linked prior argument reference" capability: a new
argument room may cite an earlier *settled* (locked) room as read-only context.
The work is well-built and design-faithful. It adds one append-only migration
(`public.argument_room_links` + RLS + two BEFORE triggers), a clean new pure-TS
module under `src/features/arguments/crossRoom/` (model, copy, caller-scoped API
client, presentational chip row), and three small additive integrations
(timeline header chip row, one-box `root_claim` draft, Inspect popout section).
The security-critical part — the access check is RLS-enforced server-side, not
UI-only — is correct: the prior room's body/nodes ride the *existing*
`debates`/`arguments` RLS under the caller's JWT, QOL-042 adds no policy on
either table, and the only denormalized field is a bounded ≤200-char title
snapshot. Locked-room immutability is structurally guaranteed — there is no code
path (RLS, trigger, client, or otherwise) that writes to `debates` or
`arguments`. Doctrine is clean throughout: no verdict/truth copy, no service-role
in client, no direct insert into `public.arguments`, no AI, soft-remove only.
All four verification commands pass and the seven new test suites cover the
design §9 plan. The ported code is complete and coherent — the full suite at
8521/303 (exactly the predicted target) is the integrity proof that nothing was
lost or duplicated in the worktree port.
