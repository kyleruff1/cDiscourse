## Summary

VG-002 promotes the existing 6-segment edge strip into a 5-layer gradient wave rail (base color band → tone wash → evidence-track texture → branch/kink stubs → active-path glow), built entirely from stacked `<View>` strips and the existing `mixHex` helper. No new dependency. The implementation matches the design's dispatch matrix line-for-line, exposes a clean BR-001 seam (`derivePlaceholderBranchKind`) that the renderer is independent of, locks the EV-001 surface (`SourceChainStatus` consumed verbatim via `summarizeArtifactsForReceiptChip`), and reuses EV-002's teal hex with a sync test. The rail is `pointerEvents: 'none'` end-to-end; per-segment a11y is intentionally delegated to the node Pressables + EV-002's chip, with a single plain-English whole-rail `accessibilityLabel` on the ScrollView. Doctrine is clean: heat is activity (never truth), saturated track is trail quality (never claim quality), and zero verdict / amplification / snake_case tokens appear in any produced string. Virtualization (`visibleSegmentSlice`) bounds rendered segments by viewport, not by message count. Typecheck, lint, and 2426/95 tests all clean.

## View-count decision (orchestrator §C)

**Path chosen: (i) Accept as-is.**

Reasoning:

1. The substantive performance guarantee is the **virtualization contract** — rendered cost is bounded by viewport width, not by message count. That contract holds: `visibleSegmentSlice` returns ≤ 50 segments at 800px viewport across a synthetic 250-segment fixture (asserted at `__tests__/railSegmentModel.test.ts:539`). At 2,500 messages or 25,000 messages the slice size does not grow.
2. The design's "330" was a back-of-envelope number conditioned on the typical mix. The dominant per-segment state in a real room (`main + no_source + cool + non-active`) renders **7** Views, not 10. The typical-case bound test (`__tests__/railSegmentModel.test.ts:545`) enforces ≤ 330, and the implementer documented the absolute ceiling (50 segments × 16 layers = 800 Views) for the pathological all-active + all-broken + warm input.
3. Tightening per-segment layer count below the design's 5-layer Z stack would either (a) merge tone wash and evidence track (collapsing two doctrinally-distinct signals into one — risks readers projecting truth onto a tone), or (b) drop the dotted-broken sub-strips back to a single overlay (loses the visual reinforcement EV-002's broken popover pairs with). Both costs outweigh the absolute-worst-case View-count delta.
4. Tightening the visible-slice budget below ~33 (e.g., halving buffer to one half-viewport) would hurt scroll quality on the dominant phone landscape viewport. Not worth it.
5. The test suite already documents both bounds — typical (≤ 330) and worst-case (≤ 800) — so any future regression is detectable at PR time.

The contract is right; the headline number was conservative under one mix and optimistic under another. The design's intent — "legible at 250+ messages" — is met.

## Non-blocking suggestions

1. **Document the absolute-ceiling number in the design file itself.** The design says "≤ 330" but the implementer correctly identified the upper bound is ~500-800 in pathological inputs. Updating §Performance Plan to read "typical ≤ 330; absolute ceiling 50 × 16 = 800 Views" would prevent future reviewers from re-litigating §C. Not required — the test file already documents both bounds.
2. **Consider rounding `transformRotateZDeg` to fewer significant figures in the wrapper geometry.** Today `baseAngle = Math.atan2(dy, dx) * (180 / Math.PI)` produces full-precision floats that flow through `inputCacheKey` via `toFixed(1)` on x/y but not on the rotation. Not a correctness issue; just a tiny cache-hit-rate improvement on horizontal-vs-near-horizontal edges where atan2 may produce subtly different rotation values across renders. Defer to a perf-pass card if it ever becomes visible in profiles.
3. **Consider a `useReduceMotion()` hook in `src/lib/`** as a tiny pure-TS wrapper over `AccessibilityInfo.isReduceMotionEnabled` + a state subscription, so VG-004 / QOL-016 don't each invent their own. Defer to whichever animation card lands first.
