### Adjudication — modified `inRoomNoRoute.test.ts`

The pre-existing TL-003 test was adapted, not weakened. Findings:

- The change touches **one** `it` block (`the ArgumentTimelineMap callbacks are wired to local
  state setters`). Its original single-line regex `onJumpLatest=\{[^}]*setActiveMessageId\(...\)\}`
  genuinely could not match the new code: IX-004 expanded the inline arrows into multi-line block
  bodies, and `[^}]*` halts at the first `}`. The regex was *broken by the production change*, not
  loosened to hide a regression.
- The replacement is a 220-char window scan anchored on `onJumpLatest=` / `onJumpToRoot=`. Verified
  the window cleanly captures each full block body and nothing else (the `onJumpLatest` window ends
  exactly where `onJumpToRoot={() =>` begins; the `onJumpToRoot` window ends at its closing `}}`).
  It still fails if `setActiveMessageId(` is removed from either callback — the assertion's intent
  ("callbacks are state-wired") is intact.
- Crucially, the modified `it` was never the no-routing assertion. The actual "no routing"
  guarantee in that describe block is the **unchanged** sibling test
  (`no navigation call sits in the ArgumentTimelineMap callback path` — scans 1500 chars for the
  routing-call patterns), plus the unchanged top-level `it.each` that scans every in-room file for
  routing imports and calls. A real routing regression in the timeline map callbacks would still be
  caught. IX-004 also ships its own `timelineReadoutNoRoute.test.ts` adding behavioural + static
  no-route coverage. The test was correctly adapted to the new code shape.

### Adjudication — test approach (no react-test-renderer)

The 6 new files use pure-model tests + source scans, not runtime RTL rendering, consistent with
the documented repo discipline (the suites' own comments cite `argumentReplySidecar.test.tsx` /
`composerDockInRoom.test.ts` — the pinned `react-test-renderer` is outside `@testing-library`'s
peer range, so the repo has no working RTL infra). For this card the approach is adequate: the
panel is a thin presentation layer whose entire behaviour is expressed in the pure
`TimelineSelectedReadoutViewModel`, which is exhaustively tested; nav/stale/shared-selection wiring
is locked by anchored source scans against `ArgumentGameSurface.tsx`; no-route and ban-list are
both behavioural (model-driven) and static (source-scan). This matches the bar set by prior cards
in this run and is accepted.
