## Suggestions (non-blocking)

1. **`isHidden` upstream wiring.** `buildSectionWhatThisMoveSays` hard-codes `isHidden = false` with a comment that it would be lifted from a future surface-model field. The design (edge case #10) calls for `isHidden` to flow from the upstream surface model. SC-003 currently exposes the slot but cannot populate it — a future card (or a small follow-up) should add the surface-model field and thread it through. Not blocking because the design explicitly defers it ("upstream `is_deleted` field would be lifted from a future field on the surface model").
2. **Boundary scan symmetry.** The model test has the SC-004 forbidden-import scan; the component test has its own scan but uses slightly different regexes (e.g., the component test allows `actionChip[^L]`). Consider unifying into a single shared `FORBIDDEN_SC004_PATTERNS` constant so they can't drift. Cosmetic.
3. **R2 dedup edge case.** Test §7 "R2 dedup: manual tag wins over auto-metadata with identical label" can't construct a real collision because the current RULE-003 maps don't produce one. The test instead asserts ordering. That's fine, but a synthetic dedup test (e.g., monkey-patching `getManualTagUx` / `getAutoMetadataUx` via `jest.spyOn`) would more directly prove the policy. Cosmetic.
4. **`getMetadataForMove` dead-code.** Line 672 of the model has `void getMetadataForMove(input.metadataLedger, node.messageId);` with a comment about forward-compat for ST-002 / HIST-001. The `void` call is harmless but slightly ugly; a `// eslint-disable-next-line @typescript-eslint/no-unused-vars` on the import + a TODO marker would be cleaner. Cosmetic.
