### Suggestions (non-blocking)

1. **`PlainHeader label="Debate"` in `DebateListScreen.tsx:297`.**
   The legacy not-mounted `DebateListScreen.tsx` (rendered behind
   `{false && ...}` in `App.tsx`) still has a single-word `label="Debate"`
   column header. The audit's `classifyText` does not flag a bare
   single-word token (line 98-105 of `uxTerminologyAudit.test.ts` documents
   this as intentional: bare tokens are treated as ban-list constants / enum
   values, not UI copy). So the audit currently reports 0 across all
   categories. But this column header IS user-visible if the screen is ever
   re-mounted, and `__tests__/debateListSort.test.ts:22` explicitly asserts
   `label="Debate"` exists. Two reasonable follow-ups:
   (a) leave it — the screen is legacy not-mounted and the live Arguments
       tab uses `ConversationGalleryScreen`; OR
   (b) bump the column header to `label="Argument"` and update
       `debateListSort.test.ts:22` to match.
   This is NOT a blocker for QOL-035 — the audit is the acceptance gate and
   it returns 0. Flag as a tiny follow-up card (e.g. QOL-035.1) if the
   `DebateListScreen` ever comes off the legacy shelf.

2. **`accessibilityLabel="debates-table"` and sibling testIDs in
   `DebateListScreen.tsx`** (line 291-293): these are
   accessibility-label / testID strings that screen readers may announce.
   "debates-table" reads aloud as "debates table". The audit doesn't flag
   them because they aren't in a sentence and they look like programmatic
   identifiers. Same legacy-not-mounted disposition as suggestion 1.

3. **`docs/core/current-status.md` entry is unusually long** (35 lines
   for what is a pure-string-substitution card). Not a blocker — the
   current-status.md ledger is the canonical surface and detailed entries
   are the established pattern. Just noting for awareness.
