## Summary

QOL-033 ships the **Go popout** — the navigate-and-re-view surface, the third
popout on the QOL-030 chassis beside Act and Inspect. It adds a pure-TS content
model (`goPopoutModel.ts`) that produces the four design §3.2 control groups
(Jump · View · Density · Lens) with each entry's active/disabled state resolved,
and a render component (`GoPopout.tsx`) that stacks those groups on the existing
`Popout` / `PopoutGroup` chassis and embeds the IX-002 mini-map read-only. The
work is exemplary: the pure model is deterministic and side-effect free, the
design §3.3 lens stage tables are verified *through* IX-001's real
`applyTimelineLens` (not re-derived), all §6 edge cases have tests, the doctrine
is respected to the letter, and the file footprint is exactly the 6 expected
files with the QOL-030 chassis untouched. Verification battery is fully green
and the test count lands exactly on the implementer's reported 8041/293. No
blockers, no doctrine concerns. One non-blocking observation about host wiring.

## Suggestions (non-blocking)

1. `GoPopout.tsx` is a complete, standalone render component but is not yet
   mounted in `OneBox.tsx` (no `goPopoutVisible` state / `openGoPopout`
   handler). This mirrors the current Act pattern — `OneBox.tsx` consumes
   `buildActPopout` the model directly and there is no separate mounted
   `ActPopout` component either — so this is consistent with the repo and the
   card scope (the card delivers the Go popout surface; host integration is a
   separate concern the design never mandated). Flagging only so a future card
   (or the QOL-030 follow-up) explicitly wires the Go entry point into the
   board chrome.
2. The Evidence-lens test covers the `source_requested · quote_requested`
   debt stages directly and explicitly defers `sourced` to IX-001's own
   coverage. This is honest and correct (Go consumes IX-001 read-only), but a
   one-line cross-reference comment naming the IX-001 test that owns the
   `sourced` case would make the delegation easier to audit later. Optional.
