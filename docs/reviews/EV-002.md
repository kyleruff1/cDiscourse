## Summary

EV-002 ships the first visible-UI surface for the evidence object model: a
pure-TS dispatch model that maps every `SourceChainStatus` to one
non-accusatory popover action, two RN components (`ReceiptChip` and
`SourceChainPopover`), and targeted wiring through the existing argument
room — `TimelineNodePopover`, `ArgumentTimelineMap`, `ArgumentGameSurface`,
`ArgumentTreeScreen`, `ArgumentComposer`, `quickActionPresets`,
`conversationMoves`, `argumentGameSurfaceModel`. EV-001's surface is
consumed verbatim — `SourceChainStatus`, `ReceiptChipContract`, and
`TimelineEvidenceContract` are imported, never redefined; the popover
headlines come from `RECEIPT_CHIP_COPY` byte-for-byte. Every "ask" CTA is
question-shaped; every preset body uses inspection / trail language.
Observer-mode contract (`isReadModeViewer === true` → visible but disabled
+ "Join a side to ask") and own-message contract (`isOwnMessage === true`
→ ask CTAs hidden across every status) are both implemented as locked.
No migration, no Edge Function, no Supabase write, no service-role, no
production AI call. The implementation is doctrine-clean and ready for
the operator to push and PR.

- [x] **Popularity is never evidence** — amplification-token ban-list (`likes` / `retweets` / `shares` / `views` / `followers` / `verified` / `engagement` / `amplification` / `trending` / `virality` / `popular` / `viral`) runs across every produced string and every preset body; the popover model consumes only `chip.status` + `chip.count` (test `buildSourceChainPopoverModel reads only chip.status / chip.count from EV-001 (no engagement metrics)` enforces the exposed surface)
- [x] **Popover never accuses the author** — every "ask" CTA label begins with "Ask for" (asserted by `/^ask for /`); preset bodies are second-person questions ("Could you point to…", "Could you quote…", "This trail looks weak — … Is there a more primary record we could anchor on?"); banned person-labels (liar / dishonest / bad faith / manipulative / extremist / propagandist / troll / astroturfer) are explicitly tested against
- [x] **Status describes the trail, not the truth** — every "ask" label asks; read-only states use the neutral verb "Inspect"; truth phrases (`is true`, `is false`, ` true.`, ` false.`) are explicitly banned by a separate assertion that runs across every produced string per status
- [x] **Epic-specific doctrine — Evidence**:
  - The popover consumes only the EV-001 chip contract — no engagement metric path exists in the popover model (cdiscourse-doctrine §3, evidence-doctrine "What does NOT count as evidence")
  - The "ask" composer flow seeds a body but routes through the existing `submit-argument` Edge Function and `evaluateArgumentDraft` — no bypass (evidence-doctrine "How challenges create debt" + "Repair moves")
  - Read-only states (`source_and_quote`, `primary_present`) carry no composer trigger — observer of a strong trail is not pushed into a redundant ask (matches the "engagement credit and factual-standing credit are separate scores" rule)
  - The `broken` headline reuses EV-001's locked "Source trail is weak" — the design's "trail language, not author language" rule is honored

**Accepted test-pattern adaptation**: the design specified `@testing-library/react-native` render-tree component tests for `ReceiptChip.test.tsx` and `SourceChainPopover.test.tsx`. The implementer found that package was declared in `package.json` but not installed in `node_modules`, with a peer-version mismatch (`react-test-renderer` 19.2.0 vs required 19.1.0). Per the no-speculative-dep-install constraint, the implementer extracted the components' load-bearing decisions into pure helpers (`buildReceiptChipDisplayLabel`, `buildReceiptChipContainerStyle`, `buildReceiptChipAccessibilityLabel`, `RECEIPT_CHIP_HIT_SLOP`, `RECEIPT_CHIP_RING_COLOR` on the chip side; `planSourceChainPopover`, `buildSourceChainPopoverAccessibilityLabel`, `SOURCE_CHAIN_POPOVER_OBSERVER_HELPER` on the popover side) and tested those instead — same dispatch matrix, same edge cases, same ban-list. **Operator-accepted at design handoff; follow-up tracked as QOL-022 (#58)** for installing the testing library, aligning the renderer version, and adding the originally-specified render-tree tests.
