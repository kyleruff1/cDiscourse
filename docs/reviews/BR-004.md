## Summary

BR-004 ships a deterministic pure-TS branch-grammar layer that sits between BR-001's
topology classification and the timeline renderer. Two new files —
`branchGrammarModel.ts` (the `BranchDirection` grammar enum + `deriveBranchDirection`
+ `buildBranchGrammarMap` + the collapsed-summary builder + the IX-004 hand-off + the
BR-003/MCP-010 advisory bridges) and `branchGrammarRenderContract.ts` (the
shape/position/stroke render-token layer) — are joined by a `BRANCH_GRAMMAR_COPY` block
in `gameCopy.ts` and a genuinely minimal, optional, back-compatible wiring step in
`ArgumentTimelineMap.tsx` + `BranchCollapseStub.tsx`. The implementation matches the
design section-for-section. Doctrine is clean: a branch direction describes a
structural position only, a chime-in cannot auto-promote to the mainline, the three
directions are distinguishable without color, and `evidence_passthrough` returns
`inherit` for every render token so evidence branches are never restyled. The one
factual divergence — MCP-010 being a design-only card with no real `SemanticOverrideLane`
export — was resolved correctly: a verbatim local mirror of MCP-010's documented
vocabulary, fully disclosed in the design-doc implementer note. Typecheck, lint, and
all 96 BR-004 tests pass. No concerns block merge.
