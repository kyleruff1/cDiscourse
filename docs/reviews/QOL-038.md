The non-web case still works post-signup if the user re-taps the link on the device (which captures into AsyncStorage on the next cold start) — the doctrine of "never trap the user" holds. Worth a follow-up card to wire `expo-linking`'s `getInitialURL()` + `addEventListener`, but this is a v1-acceptable scope reduction for a web-first deploy.
