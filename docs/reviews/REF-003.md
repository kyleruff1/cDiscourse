## Suggestions (non-blocking)
1. The teaching-state condition is `refereeObservations.length === 0 && relationToParent === 'replies'`.
   This is faithful to the design, but it couples the empty state to the `replies` fallback
   sentinel. If REF-002 ever renames or adds a second neutral relation, the teaching state
   silently stops firing. Consider a future REF-002-owned `isNeutralFallback` boolean on the
   contract so the surface doesn't re-encode the sentinel. Defer-able; not in scope for REF-003.
2. `handleRefereeMove` casts `quickAction as QuickActionLabel`, matching the existing
   `handleActDockAction` pattern (line ~1439). If `actEntryToQuickAction`'s return type is
   later narrowed to `QuickActionLabel | null`, the cast becomes removable repo-wide. Cosmetic.
