# Conversation Gallery + Horizontal Timeline UX

_Stage 6.3 — UI / model only. No xAI, no Anthropic, no X API, no Supabase write, no service-role, no DB migration._

## What this is

Replaces the row-per-debate sortable table with a **card-per-conversation** gallery. The repeated `[xai-adv …]`, `[ai-corpus …]`, `[stress …]` rooms generated by the test corpus runners are visually collapsed into a single card per canonical conversation/thread family, with the duplicate run count exposed inline.

Also patches the in-room timeline so the primary navigation rail is a **horizontal baseline**: the first child of a parent continues on the parent's lane; additional siblings branch above/below. Deep chains no longer scatter diagonally.

## Pure model

`src/features/debates/conversationGalleryModel.ts` is pure TypeScript (no React, no Supabase). It exports:

- `ConversationGalleryCard` — the per-conversation projection.
- `ConversationGalleryBucket`, `BUCKET_DEFINITIONS` — bucket id + label + empty copy.
- `ConversationHeatLevel` — `cold | warming | hot | overheated`.
- `ConversationTemperament` — `plain | curious | sharp | pedantic | evidence_heavy | source_chain_heavy | chaotic | near_resolution`.
- `ConversationSignal`, `ConversationSortMode`, `ConversationDedupeMode`.
- `buildConversationGalleryCards()` — turns `Debate[]` + per-debate `argumentsByDebateId` into cards.
- `dedupeConversationCards(cards, mode)` — visually collapses duplicates by `canonicalConversationKey`.
- `classifyConversationBucket(args)` — bucket classifier from heat / axis hits / temperament.
- `computeConversationHeat(args)` — deterministic heat score in [0,1] with level mapping.
- `computeConversationTemperament(args)` — `near_resolution`, `chaotic`, `source_chain_heavy`, etc.
- `getConversationSignals(card)` — short chip strip for the card header (no-rebuttal, source-chain pressure, etc.).
- `getConversationSearchText(card)` — pre-derived lowercase search blob.
- `sortConversationGalleryCards(cards, mode)` — six sort modes.
- `paginateConversationGalleryCards(cards, pageSize, pageIndex)` — clamps page index.

## Dedupe behavior

**Visual only.** Database rows are untouched.

`deriveCanonicalConversationKey` picks the first available key from:

1. Explicit `hintHash` (sourceHash / scenarioId, when callers have one).
2. Cleaned title (`cleanTitleForDedupe` strips `[xai-adv …]` / `[ai-corpus …]` / `[stress …]` / `[scenario …]` / `[run …]` suffixes) + normalised root body excerpt.
3. Cleaned title + normalised resolution.
4. Resolution + debate id (no collapse, but still produces a stable key).

The "primary" card for a group is the one with the latest activity; older variants collapse into it. `duplicateDebateIds` retains the underlying debate ids so an expandable run history is possible.

## Buckets

| Bucket id | Label | Trigger |
|---|---|---|
| `needs_rebuttal` | Needs rebuttal | root posted, 0 rebuttals |
| `gaining_heat` | Heating up | warming + at least one challenge |
| `hot_now` | Hot now | heat level hot or overheated |
| `source_chain_fight` | Source-chain fights | source-chain axis / `ask_source` tags / `platformSupportWarning` |
| `evidence_fight` | Evidence fights | evidence axis ≥ 2 or `evidentiaryRisk` high |
| `definition_scope_fight` | Definition / scope | definition + scope tag hits ≥ 3 |
| `pedantic_plain` | Quiet / pedantic | temperament `pedantic` or `plain` |
| `unresolved_deep_chain` | Unresolved deep chain | max depth or 8+ moves without stop reason |
| `resolved_or_synthesized` | Resolved | stop reason matches `synthesis` / `concession` |
| `my_rooms` | Mine | user has joined or authored |
| `all_open` | All | fallback |

## Heat model

Inputs (deterministic, no popularity / virality):

- Latest-activity recency (tiered: <1h, <6h, <1d, <3d, older).
- Move count.
- Rebuttal count.
- Participant count.
- Source-chain hits.
- Evidence hits.
- Challenge run length.
- Hostile tone hits (from `ad_hominem` / `civility_risk` flags).
- `platformSupportWarning` adds friction, **not** truth credit.

Score is clipped to [0,1]; level mapping:

- `cold` < 0.22
- `warming` < 0.50
- `hot` < 0.78
- `overheated` ≥ 0.78

## Temperament

| Temperament | Trigger |
|---|---|
| `near_resolution` | synthesis ≥ 1 OR concession ≥ 2 |
| `chaotic` | hostile ≥ 3 OR challenge run ≥ 5 |
| `source_chain_heavy` | source hits ≥ 2 and ≥ evidence hits |
| `evidence_heavy` | evidence hits ≥ 2 |
| `pedantic` | definition + scope ≥ 3 AND no evidence |
| `sharp` | challenge run ≥ 2 |
| `curious` | move count ≥ 3 |
| `plain` | fallback |

## UI

`src/features/debates/ConversationGalleryScreen.tsx`. App.tsx routes the Arguments tab (when no room selected) through this component.

Layout:

```
HEADER: "Arguments" / "Find a room to join"
CONTROLS: [ search ……………………… ] [ + New room ]
BUCKET CHIPS: [Any] [Needs rebuttal] [Heating up] [Hot now] [Source-chain] [Evidence] …
SORT CHIPS:   [Latest activity] [Newest created] [Heat] [Needs rebuttal first] [Most moves] [Oldest unresolved]
COUNT ROW:    "32 rooms · 11 duplicate runs collapsed"   [12 / 24 / 48]
CARDS:        responsive list of ConversationCard
PAGER:        ‹ Prev · Page N of M · Next ›
```

Each card shows: bucket headline, heat pill, temperament pill, title (with cleaned suffix), starter label, duplicate count if collapsed, FIRST POST excerpt, LATEST post excerpt + author, the `ConversationMiniTimeline` rail, moves / replies / participants stats, signal chips (no-rebuttal, source-chain warning, etc.), and an action label that swaps between `Continue →` / `Tap to join →` / `Open →`.

## Mini timeline

`src/features/debates/ConversationMiniTimeline.tsx`. One dot per posted move on a single horizontal baseline (no diagonal scatter). Tinted underlays mark detected runs (`first_clash`, `evidence_run`, `hot_zone`, `source_chain_run`). End markers: red `!` for unresolved, purple `★` for resolved.

The band-grouping helper (`groupSegmentsIntoBands`) is exported as a pure function and unit-tested.

## Full timeline rail (in-room)

`ArgumentTimelineMap` already used a horizontal x-axis. Stage 6.3 tightens lane computation so chains stay on a single horizontal line until they fork:

- **First child of a parent → same lane as parent** (continues the chain).
- **Additional siblings** branch off; odd index above, even index below.
- Lane magnitude clamped to ±`TIMELINE_MAX_LANES / 2`.

Previous parity-driven assignment (`even sibling → above`) made every reply nudge to a new lane, producing the diagonal scatter pattern. The new logic keeps the rail horizontal until there's a genuine branch.

## Data loading

`src/features/arguments/argumentsApi.ts` adds `listArgumentsForDebateIds(ids, limit=1500)` — a single `.in('debate_id', ids)` query for the gallery's batched needs. RLS still gates row visibility; no service-role.

`src/features/debates/useGalleryArguments.ts` wraps it as a React hook with stable cache-keying on the sorted debate id list and a `refresh()` for manual reloads.

App.tsx feeds the hook's `argumentsByDebateId` into the gallery screen so cards have first/latest excerpts and stats without N+1 calls.

## Future voting placeholder

Cards reserve three fields that are intentionally `null` / `0` for now:

- `voteScorePreview?: null`
- `winnerPreview?: null`
- `promotedArgumentCount?: 0`

Voting UI is not implemented in this stage. Cards have visual room for a `promoted argument`, a weighted score, and a winner badge whenever the constitution + scoring pipeline lands.

## Hard constraints upheld

- No xAI / Anthropic / X / OpenAI calls from app code (Stage 6.0 rule still binds; gallery is pure UI/model).

2. Confirm the gallery renders cards (not the old sortable table).
3. Confirm repeated `[xai-adv …]` / `[ai-corpus …]` / `[stress …]` rooms collapse into one card with `N duplicate runs collapsed` shown.

5. Tap bucket chips — pagination resets, filter applies.
